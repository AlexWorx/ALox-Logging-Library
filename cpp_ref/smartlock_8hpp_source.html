36 /** ************************************************************************************************
37  * This class extends class ThreadLock, adding functionality to register 'acquirers' of type
38  * \b %ThreadLock. Only with the second \e acquirer added, the lock is activated
39  * using \ref aworx::lib::threads::ThreadLock::SetSafeness "ThreadLock::SetSafeness(Safeness::Safe)".
40  * The goal is to not use a mutex, when such use is not needed. In occasions with very high
41  * frequency of acquisition, this can provide a performance benefit.
42  *
43  * <b>The following rules apply:</b><br>
44  * - An instance of this type must not be acquired before an \e acquirer is registered.
45  * - The \e acquirers have to be in recursive mode.
46  * - If \e acquirers are locked in a nested fashion, then they have to be added
47  * in the same order they are locked and removed in reverse order
48  * - An \e acquirer must not be added twice. (This is not a technical restriction, but a chosen
49  * design. While a second addition is ignored, in debug versions of the code, an
50  * <em>ALib Error Report</em> is written (by default this triggers an assertion).
51  *
52  * <b>Using nulled acquirers:</b><br>
53  * Sometimes it is useful to add a \c nullptr as an \e acquirer. A sample for this is found and
54  * explained with
55  * \ref aworx::lib::ALib.StdOutputStreamsLock "ALib.StdOutputStreamsLock".
56  * If the first acquirer is nullptr, the second should be added in a thread-safe way. This means,
57  * the code invoking #AddAcquirer needs to care for itself, that this object is not acquired
58  * during this process. E.g. it can be done in the bootstrap section of a process, when no parallel
59  * threads were started. For further acquirers, such care does not need to be taken.
60  * While an \e acquirer must not be attached twice, 'anonymous' (nullptr) \e acquirers may.
61  * For each anonymous invocation of #AddAcquirer, a corresponding call #RemoveAcquirer is
62  * needed, to get back to \b Safeness::Unsafe.
63  **************************************************************************************************/
