132  /** ****************************************************************************************
133  * This method needs to be invoked when values of type \ref Ticks are to be converted to "real"
134  * clock time values. Whenever the system clock is changed, such change is reflected by class
135  * \b %Ticks and its conversion methods
136  * \alib{time,Ticks::InEpochSeconds},
137  * \alib{time,Ticks::SetFromEpochSeconds},
138  * \alib{time,Ticks::SetFromEpochSeconds},
139  * \alib{time,Ticks::SetFromEpochSeconds},
140  * \alib{time,Ticks::SetFromEpochSeconds} and
141  * \alib{time,Ticks::SetFromEpochSeconds},
142  * <b>only after</b> the invocation of this method.<br>
143  *
144  * The advantages of this approach are:
145  * - As long as this method is not called, changes of the system clock do not affect the display
146  * of time values measured with class \b %Ticks. They will continue to display clock time values
147  * which are in alignment with the system clock setting as it was when this method was invoked
148  * last (e.g. the bootstrap of the process).
149  * - Round-trip calculations from \b %Ticks to clock times and back are consistent when this method
150  * is not invoked.
151  *
152  * Of-course, the drawback is: to ensure that the actual clock setting is reflected, the method
153  * has to be invoked 'manually' prior to using one of the methods listed above.
154  *
