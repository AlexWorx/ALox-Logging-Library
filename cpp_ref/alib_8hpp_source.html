404  /**
405  * This is a smart mutex that allows to lock an applications' <em>standard output
406  * streams</em>.
407  *
408  * In multi-threaded processes, to protect the output streams from concurrent access,
409  * this smart lock might be used by any \e entity that writes data to the streams.
410  * Before it can be used (acquired and released), it is needed to register with the object
411  * using
412  * \ref aworx::lib::threads::SmartLock::AddAcquirer "SmartLock::AddAcquirer".
413  * This has to be done once per thread that aims to write to the stream. Then, prior to
414  * writing, this object has to be acquired and after writing released.
415  *
416  * Because often, the standard \e output stream and standard \e error stream are identical,
417  * ALib provides one single lock for both, to protect also against interwoven
418  * standard output and error information.
419  *
420  * If the 'entity' that is registering is not of type
421  * \ref aworx::lib::threads::ThreadLock "ThreadLock" it is allowed to provide \c nullptr
422  * in the parameter of method \b AddAcquirer. In this case, the process of adding and
423  * removing \e acquirers is not performed in a thread safe way. Therefore it is advised
424  * to register so called anonymous (\c nullptr) \e acquirers only at bootstrap time, when
425  * no parallel threads were started, yet.
426  *
427  * If an application is deemed to always write to the standard output streams from within
428  * multiple threads, an alternative to registering each writing entity, is to
429  * invoke \b AddAcquirer just two times in a row with \c nullptr at the start of a process
430  * and then never do this again (and never de-register). This way, no thread needs
431  * to register/deregister but threads may still \b Acquire and \b Release the lock without
432  * being registered. In other words, once a smart lock is enabled, subsequent registrations
433  * are just used to count and identify the de-registration.
434  *
435  * \note
436  * The advantage of the SmartLock is that if only one 'entity' registered, no
437  * system \e mutexes will be used with \b Acquire and \b Release, hence there is
438  * a performance gain. Such gain is not noticeable for the 'slow' terminal console output,
439  * but it is for fast, buffered output streams.
440  * <p>
441  * \note
442  * Logging library \b ALox, which is built on ALib, will register whenever a \e Logger
443  * is used that writes to the standard output stream. Hence, applications that in
444  * parallel use, e.g. 'std::cout', should register at bootstrap and \e acquire this
445  * instance prior to writing. This way, log output and other application output is
446  * not mixed, but separated in different Lines.
447  * <p>
448  * \note
449  * This fields is available only in the complete distribution version of \b %ALib.
450  */
451  #if ALIB_MODULE_ALL
452  ALIB_API static threads::SmartLock StdOutputStreamsLock;

515  /** ****************************************************************************************
516  * This method must be called prior to using ALib, e.g. at the beginning of
517  * the \c main() method of an application. It is OK, to call this method more than once, which
518  * allows independent code blocks (e.g. libraries) to bootstrap %ALIB independently.
519  * However, only the first invocation is effective with the exclamation that if
520  * command line parameters are provided with a call, those are set.
521  * Also, the very first invocation should not be interrupted by a parallel invocation of a
522  * second thread. Consequently, it has to be assured that this method is invoked once on
523  * the real bootstrap an app.
524  *
525  * The following actions are performed in the full distribution version of \b ALib. With
526  * different <b>ALib Module</b> distributions, none-applicable steps are omitted.
527  * - The configuration object is created
528  * - Classes of ALib namespace \ref aworx::lib::threads are prepared to work properly
529  * - Classes of ALib namespace \ref aworx::lib::time are prepared to work properly
530  * - glibc versions of ALib (GNU/unix) probably invoke glibc method
531  * <em>setlocale()</em>, depending on the setting of the environment variables
532  * <em>LANG</em> and <em>LANGUAGE</em>
533  * and depending on ALib configuration variable
534  * [ALIB_LOCALE](../group__GrpALoxConfigVars.html).
535  * - Config variable [WAIT_FOR_KEY_PRESS](../group__GrpALoxConfigVars.html)
536  * is read and field #WaitForKeyPressOnTermination set accordingly
537  *
538  * \note If other, custom configuration data sources should be used already with this method
539  * (to read the configuration variables as described above),
540  * according configuration plug-ins have to be added attached to public, static field
541  * \ref aworx::lib::config::Configuration::Default "Configuration::Default"
542  * prior to invoking this method.
543  *
544  * <p>
545  *
546  * \note %ALIB must not be used before all global/static variables are created. Hence, it
547  * is not allowed to initialize %ALIB within static variable initialization code.
548  * This restriction is wanted by design, because different platforms and compilers
549  * implement the initialization phase of static and global code differently and it is
550  * not considered good programming style to rely on C++ bootstrap.
551  * Using %ALIB within C++ bootstrap is undefined behavior.
552  *
553  * <p>
554  *
555  * \note On the Windows platform, the Microsoft compiler provides the global variables
556  * <em>__argc</em> and <em>__argv</em> (respectively <em>__wargv</em> for wide
557  * character binaries. These variables a can be used if this method is invoked
558  * outside of the <em>main()</em> method.
559  *
560  * @param argc Parameter usually taken from <em>standard C</em> \c main() method
561  * (the number of arguments in \p argv).
562  * Defaults to 0.
563  * @param argv Parameter usually taken from <em>standard C</em> \c main() method
564  * (pointer to a list of command line arguments).
565  * Defaults to nullptr.
566  ******************************************************************************************/
