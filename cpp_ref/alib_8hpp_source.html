229 /**
230  * @defgroup GrpALibCompilerSymbols ALib Compilation Symbols
231  *
232  * Symbols (C++ preprocessor macros) listed here, support adding or removing features to \b %ALib when
233  * compiling the library and any code unit that uses \b %ALib.
234  * These symbols must be used <em>only within makefiles or IDE project files to be passed to the
235  * C++ compiler (!) </em> (normally -D option). In other words: they must not be defined or undefined
236  * within the source code that uses \b %ALib.
237  *
238  * \b %ALib Symbols are prefixed with <em>ALIB_</em>.
239  *
240  * For most of the symbols, a version with suffix <em>_ON</em> and for some also one with the suffix
241  * <em>_OFF</em> exists. These symbol decide about the definition, respectively the avoidance of the
242  * definition of a corresponding <em>code selection symbol</em> that is named the same without the
243  * suffix.<br>
244  * If both are set at the same time, a compiler error is generated by \b %ALib header files.
245  * If none of them is passed to the compiler, a default value is assumed.<p>
246  *
247  * As a convenience, to get default behavior, no symbol needs to be passed to the compiler.<p>
248  *
249  * \note Some of the compilation symbols are changing the definition of classes and methods which
250  * makes binaries incompatible with binaries that do not use the same setting for each of
251  * these symbols.<br>
252  * Symbols that have to be kept equal across compilation units are denoted in this
253  * documentation.
254  * @{ @}
255  */

469  /**
470  * This is a smart mutex that allows to lock an applications' <em>standard output
471  * streams</em>.
472  *
473  * In multi-threaded processes, to protect the output streams from concurrent access,
474  * this smart lock might be used by any \e entity that writes data to the streams.
475  * Before it can be used (acquired and released), it is needed to register with the object
476  * using
477  * \ref aworx::lib::threads::SmartLock::AddAcquirer "SmartLock::AddAcquirer".
478  * This has to be done once per thread that aims to write to the stream. Then, prior to
479  * writing, this object has to be acquired and after writing released.
480  *
481  * Because often, the standard \e output stream and standard \e error stream are identical,
482  * \b %ALib provides one single lock for both, to protect also against interwoven
483  * standard output and error information.
484  *
485  * If the 'entity' that is registering is not of type
486  * \ref aworx::lib::threads::ThreadLock "ThreadLock" it is allowed to provide \c nullptr
487  * in the parameter of method \b AddAcquirer. In this case, the process of adding and
488  * removing \e acquirers is not performed in a thread safe way. Therefore it is advised
489  * to register so called anonymous (\c nullptr) \e acquirers only at bootstrap time, when
490  * no parallel threads were started, yet.
491  *
492  * If an application is deemed to always write to the standard output streams from within
493  * multiple threads, an alternative to registering each writing entity, is to
494  * invoke \b AddAcquirer just two times in a row with \c nullptr at the start of a process
495  * and then never do this again (and never de-register). This way, no thread needs
496  * to register/deregister but threads may still \b Acquire and \b Release the lock without
497  * being registered. In other words, once a smart lock is enabled, subsequent registrations
498  * are just used to count and identify the de-registration.
499  *
500  * \note
501  * The advantage of the SmartLock is that if only one 'entity' registered, no
502  * system \e mutexes will be used with \b Acquire and \b Release, hence there is
503  * a performance gain. Such gain is not noticeable for the 'slow' terminal console output,
504  * but it is for fast, buffered output streams.
505  * <p>
506  * \note
507  * Logging library \b %ALox, which is built on \b %ALib, will register whenever a \e Logger
508  * is used that writes to the standard output stream. Hence, applications that in
509  * parallel use, e.g. 'std::cout', should register at bootstrap and \e acquire this
510  * instance prior to writing. This way, log output and other application output is
511  * not mixed, but separated in different Lines.
512  * <p>
513  * \note
514  * This fields is available only in the complete distribution version of \b %ALib.
515  */
516  #if ALIB_MODULE_ALL
517  threads::SmartLock& StdOutputStreamsLock;
518  #endif
