ALIB_API bool GetEnvironmentVariable ( const TString &  varName,
AString &  target,
lang::CurrentData  targetData = lang::CurrentData::Clear,
lang::Case  sensitivity = lang::Case::Sensitive 
)

Returns the value of an environment variable.

While under posix compatible operating systems, environment variable names are case sensitive, under Microsoft windows they are not. This behavior is reflected by the default value of parameter sensitivity. Its options are as follows:

  • GNU/Linux
    Defaults to Cases::Sensitive. Can be changed by passing Case::Ignore.
  • Windows OS
    Defaults to Cases::Ignore. A run-time assertion is thrown when Case::Sensitive is passed, because this is not supported. In release compilations, the parameter is ignored.

  • Mac OS
    Defaults to Cases::Sensitive. An compile time assertion is given when Case::Ignore is passed, because this is not supported. In release compilations, the parameter is ignored.

    Parameters
    varNameThe name of the variable to be retrieved.
    targetAn AString to write the result to.
    targetDataIf CurrentData::Keep, the parameter target is not cleared before the result is written. Defaults to CurrentData::Clear.
    sensitivityUse case sensitive mode for variable names. Not supported under Microsoft Windows OS. Defaults to Case::Ignore.
    Returns
    true if variable was found.
