70  * \b %Ticks and its conversion methods

72  * \ref aworx::lib::time::Ticks::SetFromEpochSeconds "Ticks::SetFromEpochSeconds",

77  * invocation of this method.<br>
78  * The advantages of this approach are:
79  * - As long as this method is not called, changes of the system clock do not affect the display
80  * of time values measured with class \b %Ticks. They will continue to display clock time values
81  * which are in alignment with the system clock setting as it was when this method was invoked
82  * last (e.g. the bootstrap of the process).
83  * - Round-trip calculations from \b %Ticks to clock times and back are consistent when this method
84  * is not invoked.
85  *
86  * Of-course, the drawback is: to ensure that the actual clock setting is reflected, the method
87  * has to be invoked 'manually' prior to using one of the methods listed above.
88  *
89  * The method is invoked by \ref Init.
90  */

93 /** ************************************************************************************************
94  * This class provides an interface into system dependent timer values (usually 64 bit
95  * tick counters) that we call *ticks*. Ticks and this class allow to store and calculate time
96  * values in an efficient and highly accurate way.
97  *
98  * Ticks generated by this class can be added and subtracted:
99  *
100  * - Subtraction generates a *span value in ticks*
101  * - *Span value in ticks* can be added to themselves safely
102  * - *Span value in ticks* can be added to ticks safely
103  *
104  * Conversion functions to milliseconds, microseconds and nanoseconds accept ticks and span
105  * values in ticks in the same manner.
106  *
107  * As Ticks are system dependent, they should not be stored and loaded in data files or
108  * otherwise shared between systems. Even on the same machine, the values may become be
109  * incompatible (e.g. after a reboot).
110  * Also, in general such values should only be used by methods of
111  * this class and classes
112  * \ref aworx::lib::time::TickWatch "TickWatch",
113  * \ref aworx::lib::time::TickSpan "TickSpan" and
114  * \ref aworx::lib::time::TicksCalendarTime "TicksCalendarTime".
115  *
116  * The resolution and accuracy of the values is platform dependent. Especially nanoseconds are
117  * deemed to be not accurate and above 100 ns (this was written and fact in 2013).
118  **************************************************************************************************/
