aworx::lib::boxing::ftypes Namespace Reference

Namespace Description

ALib Boxing allows to map C++ types to boxed types in a non-bijective way. As a consequence, there are different possibilities to implement such mapping and the mapping that ALib Boxing uses by default for boxing C++ fundamental types is considered to be just a proposal and a default behavior.

To suppress the use of this built-in implementation of boxing C++ fundamental types, compilation symbol ALIB_FEAT_BOXING_FTYPES_OFF may be passed to the compiler. This freedom of choice is limited when using the full distribution of ALib or a distribution module of ALib that includes more ALib functionality than just boxing. In these cases, the default built-in implementation of boxing fundamental types is automatically chosen, because other parts of ALib rely on the default behavior. Using ALIB_FEAT_BOXING_FTYPES_OFF with these distributions results in a compilation error.

The mapping of C++ fundamental types to boxed types is done as follows:

  • Character types char, wchar_t, char16_t and char32_t are mapped to identical boxed types.
  • Arrays of character types are mapped to boxed arrays of the respective character type, e.g char[5] → char[] or wchar_t[10] → wchar_t[]. The array length is of-course stored in the box.
  • Pointers to character types the same mapped to arrays of the respective character type, e.g char* → char[] or wchar_t* → wchar_t[]. The array length is set to 0 if either the pointer is nullptr or the first character in the string is '\0'. Otherwise, the length is set to -1. Therefore, when unboxing a character array type or implementing such type in an Interface implementation, the "true" length has to be evaluated if -1 is stored!
  • Type bool is mapped to identical boxed type.
  • All signed integer types (int8_t, int16_t, int32_t, int64_t and intGap_t) are boxed as boxed_int, which is an alias of int64_t.
  • All unsigned integer types (uint8_t, uint16_t, uint32_t, uint64_t and uintGap_t) are boxed as boxed_uint, which is an alias of uint64_t.
  • Type float and double are both boxed as double.
  • Unboxing is implemented in a strict fashion: Only the "result" types are allowed to be unboxed, hence char, wchar_t, char16_t, char32_t, bool, boxed_int, boxed_uint and double.
See also
For more information about this default implementation of boxing C++ fundamental types, see manual chapter 4.2 Boxing Fundamental Types.
Attention
In this namespace documentation, you will find specializations of templated namespace functions of namespace aworx::lib::boxing. This is of-course wrong in respect to the C++ language definition, which requires such specializations to be placed in the original namespace.
It was decided to 'trick' the documentation parser to show these specialization in this namespace, to keep namespace boxing clean and to have all specializations related to this namespace in one place.
