40 /** ************************************************************************************************
41  * Encapsulates information of the caller that can be collected. This is platform specific, in
42  * this case, C++. What we receive from the C++ preprocessor, is the source file name
43  * of the calling code, the line number within the source file name and the name of the method
44  * the call is placed in. E.g., wee do not get the class name of the object or even its instance.
45  *
46  * The method has a list of interface functions that gets such source information and some derived
47  * variants of it. In addition, thread and timer information is stored and managed.
48  *
49  * As far as possible, 'lazy' techniques come to practice with this class. This means, only
50  * the things that are queried in-between to invocations of method #Set are calculated.
51  **************************************************************************************************/
