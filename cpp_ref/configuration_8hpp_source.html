292  /** ****************************************************************************************
293  * Writes the variable to the configuration.
294  * In general, this is done by asking each of our installed plug-ins - in the order of
295  * their priority - to do so.
296  * As soon as the first plug-in returns \c true, the variable is considered written and no
297  * other plug-in is asked.
298  * This way, the variable is stored by the plug-in with the highest priority.
299  *
300  * The maximum priority to start the loop with depends on field
301  * \ref aworx::lib::config::Variable::Priority "Variable::Priority"
302  * of the given \p variable. The rules are as follows:
303  * - If the value is \alib{config,Priorities::NONE}, which is the default value of new
304  * variables or ones that were freshly declared, then prior to storing the value, the
305  * variable is (tried to be) loaded first (without actually overwriting the values).
306  * After that, one of the next two rules apply.
307  * - If the value is still \alib{config,Priorities::NONE}, which indicates that a previous
308  * load operation failed, the loop starts with \alib{config,Priorities,Priorities::DefaultValues}
309  * (and usually ends there, as standard configuration sets
310  * do not have plug-ins with lower priorities installed). In other words, newly created
311  * variables are stored in the plug-in of priority #DefaultValues (usually an
312  * \b InMemoryPlugin). This way, they are
313  * not written to external configuration files, unless the application explicitly moves
314  * such new default values to dedicated other plug-ins (e.g. on termination).
315  * - If the value is greater than \c 0, the value is used as the start of the loop.
316  * This way, an already defined variable will be stored in the same plug-in as it was
317  * found or one with a lower priority, if that plug-in does not provide writing
318  * capabilities.
319  *
320  * Consequently, as field \alib{config,Variable::Priority} is public, the behavior
321  * can be manipulated, by setting the field explicitly prior to invoking this method.
322  * Some frequent use cases shall be named here:
323  * - Setting the field to \alib{config,Priorities::ProtectedValues} allows to protect the
324  * variable value in respect to external manipulation.
325  * - By setting the field to #DefaultValues, the value becomes just a default
326  * and does not overwrite externally specified values.
327  * - Setting the field to a distinct priority value that names a user-specific configuration
328  * (vs. a system-wide configuration) to store into.
329  * - Setting the field to #ProtectedValues <c>- 1</c>, allows to store the
330  * variable just in the plug-in with highest possible priority, for example
331  * a user specific configuration is preferred to a system wide configuration)
332  * - A variable might be related to a second one. If the priority of the second one is
333  * known, that priority might be set to possibly have both variables stored in the same
334  * plug-in.
335  *
336  * The method returns the priority of the configuration plug-in that the value was written
337  * to as well as storing this value in field \alib{config,Variable::Priority}.
338  * If the result is \c 0, the variable was not written. This might only happen if
339  * - either field default plug-in of priority #DefaultValues was modified
340  * (removed or exchanged with a different plug-in that does not write the value)
341  * - or if field \p Priority of the variable was set below #DefaultValues and greater
342  * than \c 0.
343  * - The detected (!) priority was #ProtectedValues.
344  * In this case, obviously the application does not want to allow changes and writing the
345  * variable into a different plug-in has no effect.
346  * This way, such variables also do not appear in a users' configuration
347  * in the case that on program termination, new default values are copied there.
348  *
349  * Optional parameter \p externalizedValue allows to provide a string that is parsed
350  * by the storing plug-in to reset the variables' values prior to writing.
351  *
352  * @param variable The variable object.
353  * @param externalizedValue Optional externalized value string. If given, the variable
354  * is set prior to writing.
355  *
356  * @returns The priority of the configuration plug-in that the value was written to.
357  ******************************************************************************************/
358  ALIB_API
359  Priorities Store( Variable& variable, const String& externalizedValue= nullptr );
