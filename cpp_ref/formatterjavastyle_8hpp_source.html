37 /** ************************************************************************************************
38  * Implements a \ref aworx::lib::strings::Formatter "Formatter" according to the
39  * [formatting standards of the Java language](http://docs.oracle.com/javase/8/docs/api/java/util/Formatter.html).
40  *
41  * \note
42  * Inherited, public fields of parent class \b FormatterStdImpl provide important possibilities
43  * for changing the formatting behavior of instances of this class. Therefore, do not forget
44  * to consult the \ref aworx::lib::strings::FormatterStdImpl "parent classes documentation".
45  *
46  * In general, the original specification is covered quite well. The differences and specialties
47  * are:
48  * - In deviation of the Java specification, after creation, a formatter in this implementation
49  * does not produce locale aware output. Instead, number formatting is set to "computational",
50  * hence the decimal point separator is <c>'.'</c> and the grouping character <c>','</c>.
51  * As the syntax specification does not provide a feature to switch between standard and locale
52  * setting, the corresponding fields of #AlternativeNumberFormat are not used with this formatter.
53  * Instead, to enable localized output, method
54  * \ref aworx::lib::strings::NumberFormat::SetFromLocale "NumberFormat::SetFromLocale"
55  * has to be invoked on field
56  * \ref aworx::lib::strings::FormatterStdImpl::DefaultNumberFormat "FormatterStdImpl::DefaultNumberFormat".
57  * Alternatively, attributes of this object may be changed manually or by other means to reflect
58  * a desired locale.
59  * - Hexadecimal floating point output (conversion type \c 'a' and \c 'A') is not supported.
60  * - Flag \c '(', used to write negative numbers in round brackets, is not supported.
61  * - Addressing the previous argument index using \c '%<' is already allowed with the first
62  * placeholder.
63  * This Chooses the first argument. (In Java a \b MissingFormatArgumentException would be thrown.)
64  * - Flag \c '^' is an extension to the standard and denotes
65  * center-alignment - just like \c '-' in the standard denotes left-alignment.
66  * Right-alignment is the default.
67  *
68  *<p>
69  * - <b>Floating point values:</b>
70  * - If standard field type \c 's' is given together with a precision, the field is cut, even if
71  * it cuts the number somewhere. (This is just a warning and same behavior as in original
72  * specification.)
73  * - For lower case floating point format types (\c 'f', \c 'g' and \c 'e'), the values specified
74  * in attributes \b %ExponentSeparator, \b %NANLiteral and \b %INFLiteral of object
75  * \ref aworx::lib::strings::FormatterStdImpl::AlternativeNumberFormat "AlternativeNumberFormat"
76  * are used. For upper case types (\c 'G' and \c 'E'), the corresponding attributes in
77  * \ref aworx::lib::strings::FormatterStdImpl::DefaultNumberFormat "DefaultNumberFormat" apply.
78  * - Fixed point format (\c 'f' ) is not supported to use arbitrary length.
79  * See class \ref aworx::lib::strings::NumberFormat "NumberFormat" for the limits.
80  * Due to this limitation, the default number of fractional digits is not set with type \c 'f',
81  * while in Java it is set to \c 6. This is to allow higher numbers up to \c 1.e13 to be printed
82  * in non-scientific format
83  * - When both, a \p width and a \p precision is given, then the \p precision determines the
84  * fractional part, even if the type is \b 'g' or \b 'G'. This is different than specified with
85  * Java formatter, which uses \p precision as the overall width in case of types
86  * \b 'g' or \b 'G'.
87  *
88  *<p>
89  * - <b>Hexadecimal and Octal Numbers:</b>
90  * - Hexadecimal and octal output is <b>cut in size</b> (!) when a field width is given that
91  * is smaller than the resulting amount of digits of the number arguments provided.
92  * \note This implies that a value written might not be equal to the value given.
93  * This is not a bug but a design decision. The rational behind this is that with this
94  * behavior, there is no need to mask lower digits when passing the arguments to the
95  * format invocation. In other words, the formatter "assumes" that the given field width
96  * indicates that only a corresponding number of lower digits are of interest.
97  *
98  * - The number <b>grouping option</b> (<c>','</c>) can also be used with binary, hexadecimal
99  * and octal output.
100  * The types support different grouping separators for nibbles, bytes, 16-bit and 32-bit words.
101  * Changing the separator symbols, is not possible with the format fields of the format strings
102  * (if it was, this would become very incompatible to Java standards). Changes have to be made
103  * prior to the format operation by modifying field
104  * \ref aworx::lib::strings::FormatterStdImpl::AlternativeNumberFormat "AlternativeNumberFormat"
105  * which is provided through parent class \b %FormatterStdImpl.
106  *
107  * - Alternative form (\c '#') adds prefixes as specified in members
108  * - \ref aworx::lib::strings::NumberFormat::HexLiteralPrefix "HexLiteralPrefix" and
109  * - \ref aworx::lib::strings::NumberFormat::OctLiteralPrefix "OctLiteralPrefix".
110  *
111  * For upper case formats, those are taken from field
112  * \ref aworx::lib::strings::FormatterStdImpl::DefaultNumberFormat "DefaultNumberFormat",
113  * for lower case formats from
114  * \ref aworx::lib::strings::FormatterStdImpl::AlternativeNumberFormat "AlternativeNumberFormat".
115  * All defaults may be changed by the user.
116  *
117  *<p>
118  * - <b>Time and Date:</b>
119  * - In the C++ version, boxed values of type
120  * \ref aworx::lib::time::Ticks "Ticks"
121  * are applicable to conversion type \c 't'.
122  * - The following time conversion suffix characters are supported:
123  * \c 'H', \c 'k', \c 'I', \c 'l', \c 'M', \c 'S', \c 'B', \c 'b', \c 'h', \c 'A', \c 'a',
124  * \c 'Y', \c 'y', \c 'm', \c 'd', \c 'e', \c 'R', \c 'T', \c 'D' and \c 'F'
125  * - The following time conversion suffix characters are \b not supported:
126  * \c 'L', \c 'N', \c 'p', \c 'z', \c 'Z', \c 's', \c 'Q', \c 'C', \c 'j', \c 'r' and \c 'c'.
127  *
128  **************************************************************************************************/
