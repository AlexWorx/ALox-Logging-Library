Class Description

template<typename TEnum, typename TCheckEnum = typename std::enable_if<std::is_enum<TEnum>::value>::type>
struct aworx::lib::lang::T_EnumIsArithmetical< TEnum, TCheckEnum >

Introduction

With scoped enums, introduced by C++ 11, arithmetic operators are not supported. This seems a little contradiction as still enums have their underlying integer type and with a little static casting, these integer values are accessible. Also, the other way round, an enum value in C++ can be initialized by passing arbitrary integer values (even those that no elements exist for) like this: 

     auto myElement= MyEnum( 42 );

Therefore, if operators are needed, the common approaches are to either

  • define operators for each scoped enum type that need them, or to
  • provide a set of templated operator functions that handle all scoped enum types.

The first approach includes a lot of duplicate code, the second has the disadvantage that it undermines the intention of the C++ 11 language standard: The compiler would allow the operators even non-bitfield custom scoped enum types.

ALib Arithmetic Enum Operators

This extremely simple TMP struct is used to overcome the disadvantages of the approaches discussed above. The default implementation is empty and just inherits from std::false_type.

Specializations of the struct for a certain enum type should be performed using macro ALIB_LANG_ENUM_IS_ARITHMETICAL. The macro needs to be located in a header file, probably close to the enum type declaration itself, but outside of any namespace (aka in the default namespace).

With this specialization, the compiler accepts a set of templated operator functions which get conditionally selected using TMP struct std::enable_if which of-course checks for this struct to be specialized for the type in question.

The following ALib set of operators are applicable to scoped enumerations if marked arithmetic:

  • operator<
  • operator<=
  • operator>
  • operator>=
  • operator+
  • operator-
  • operator+=
  • operator-=

Each operator exists in two versions: one accepting an enum element for both operands and a second that accepts the right hand side operand of the integer type that is underlying the scoped or non-scoped enumeration.

Attention
While in this documentation the operators are appearing under namespace aworx::lib::lang , in reality they are defined in no namespace (global namespace). This was done to have the documentation of the operator functions and this struct just in one place!

Arithmetic Enums and other ALib Enum Features

This TMP struct must not be specialized in parallel with TMP struct T_EnumIsBitwise. Other ALib enum features can used in parallel:

  • T_EnumIsIteratable
  • T_EnumMetaDataDecl and corresponding related TMP structs.
