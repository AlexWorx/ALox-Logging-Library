aworx Namespace Reference

Namespace Description

This is the outer C++ namespace for all classes published by A-Worx GmbH, Germany.

Note
While the Java language strongly proposes to use the complete 'reverse domain' name and therefore Java code published by A-Worx GmbH resides in namespace com.aworx, for C# we replaced the prefix com. with cs., which leads to cs.aworx.
This is for having classes with the same names existing in C++, C# and Java not collide within the Doxygen documentation system.

As far as it is planned today, there is nothing inside this namespace but other namespaces and - and this is true only for the C++ side of things - type definitions used as shortcuts.

Type Shortcuts in the aworx Namespace

Introducing new names without good reasons should in general by avoided as it pollutes any namespace. Furthermore it should be avoided to use identifier names that are so general that conflicts are foreseeable. However, in this namespace aworx there is an exclamation. It introduces shortcut types only for the sake of making the code more readable. And, even more, those shortcuts are allowed to have quite general names. For example, the following type is defined: 

using  String =  aworx::lib::strings::AS;

The impact of this is as follows:

  • Inside header files of sub-namespaces (and their sub-namespaces) of aworx, e.g. aworx::lib and aworx::lox, these names are visible and used.
  • Inside external header files, they are not visible but might be quite shortly accessed using postfix aworx::.
  • Inside cpp files that use aworx libraries they are not visible, even if a using statement for the corresponding namespace is added

As a sample, in a source file which includes ALib and 'uses' the strings namespace by having statement 

using  aworx::lib::strings;

defined, still type alias String is not visible and therefore does not collide with any other identifier that is named "String".
