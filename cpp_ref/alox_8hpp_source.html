267  /** ****************************************************************************************
268  * This method must be called prior to using \b %ALox, e.g. at the beginning of
269  * the \c main() method of an application. It is OK, to call this method more than once, which
270  * allows independent code blocks (e.g. libraries) to bootstrap \b %ALox independently.
271  * However, only the first invocation is effective with the exclamation that if
272  * command line parameters are provided with a call, those are set.
273  * Also, the very first invocation should not be interrupted by a parallel invocation of a
274  * second thread. Consequently, it has to be assured that this method is invoked once on
275  * the real bootstrap an app.
276  *
277  * \note This method invokes
278  * \ref aworx::lib::ALIB::Init "ALIB::Init", hence no explicit initialization of
279  * underlying \b %ALib is needed.
280  *
281  * If command line parameters should be used for configuring \b %ALox, then the very first
282  * call to this method has to provide the argc and argv parameters.
283  * Subsequent calls to this method with different parameters do not change the setup.
284  *
285  * \note If other, custom configuration data sources should be used already with this method
286  * (to read the configuration variables as described in
287  * \ref aworx::lib::ALIB::Init "ALIB::Init"),
288  * according configuration plug-ins have to be added attached to public singleton
289  * \ref aworx::lib::config::Configuration::Default "Configuration::Default"
290  * prior to invoking this method.
291  *
292  * <p>
293  *
294  * \note On the Windows platform, the Microsoft compiler provides the global variables
295  * <em>__argc</em> and <em>__argv</em> (respectively <em>__wargv</em> for wide
296  * character binaries. These variables a can be used if this method is invoked
297  * outside of the <em>main()</em> method.
298  *
299  * For convenience, several \b %ALox classes call this method implicitly on construction
300  * or when using them, e.g. Lox and Logger. Therefore, when using \b %ALox, there is no need
301  * to to call this method, unless aforementioned command line parameters should be passed.
302  *
303  * \note
304  * \b %ALox must not be used before all global/static variables are created. Hence, it
305  * is not allowed to initialize \b %ALox within static variable initialization code. This
306  * restriction is wanted by design, because different platforms and compilers implement the
307  * bootstrap of static and global data differently and it is not considered a good
308  * programming style to rely on C++ bootstrap. Using \b %ALox/ALIB within C++ bootstrap is
309  * undefined behavior. (This is true for any C++ library.)
310  *
311  * \attention
312  * If release logging is used, the explicit initialization of \b %ALox by invoking
313  * this method is mandatory. While debug logging 'automatically' invokes the method
314  * hidden in the background, release logging does not.
315  * If omitted, critical errors and undefined behavior in release builds might occur
316  * in the moment when debug-logging is pruned!
317  *
318  * @param argc Parameter usually taken from <em>standard C</em> \c main() method
319  * (the number of arguments in \p argv).
320  * Defaults to 0.
321  * @param argv Parameter usually taken from <em>standard C</em> \c main() method
322  * (pointer to a list of command line arguments).
323  * Defaults to nullptr.
324  * @returns Returns \c true if \b %ALox was initialized before, \c false if not.
325  ******************************************************************************************/
326  inline static
327  bool Init( int argc =0, char** argv =nullptr )
328  {
329  if ( isInitialized ) return false;
330  ALox::checkLibraryVersions(); // this call has to stay in the header file
331  ALIB::Init( argc, argv );
332  initImpl();
333  return true;
334  }
335 
336  /** ****************************************************************************************
337  * Variant of method #Init, accepting command line arguments of type \c wchar_t.
338  *
339  * @param argc Parameter usually taken from <em>standard C</em> \c main() method
340  * (the number of arguments in \p argv).
341  * @param argv The command line parameters as \c wchar_t.

343  ******************************************************************************************/
