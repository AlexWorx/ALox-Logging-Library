119  /**
120  * This is the path for logging to the internal domain. By manipulating this
121  * <em>Log Domains' Verbosity</em>, the verbosity of \b %ALox itself can be controlled.
122  * For example, with \e Verbosity.INFO, the 'on the fly' creation of <em>Log Domains</em>
123  * are logged, which can be helpful to determine the <em>Log Domains</em> that are
124  * created by libraries and larger projects.
125  *
126  * The following sub-domains are used by \b %ALox:
127  *
128  * Sub-Domain | Description
129  * - - - - - -| - - - - - - - - - - - - - - - - - - -
130  * LGR | Used when \e %Loggers are registered, retrieved or removed from a \b %Lox and when the \e Verbosity of a <em>Log Domain</em> for a \e %Logger is changed.<br>In addition used with method \b %Lox::SetStartTime.
131  * DMN | Used when <em>Log Domains</em> are registered (on first use), when <em>Scope Domains</em> are set or removed and when <em>Domain Substitution Rules</em> are set.
132  * PFX | Used when <em>Prefix Logables</em> are set or removed.
133  * THR | Used with method \b %Lox::MapThreadName.
134  * LGD | Used with storing and retrieving <em>Log Data</em> objects.
135  *
136  * In addition, class
137  * \ref aworx::lox::ALoxReportWriter "ALoxReportWriter" logs into sub-domain
138  * <c>'REPORT'</c>.
139  *
140  * \note For internal logging an separated <em>domain tree</em> is used. This means, that
141  * setting the root domain of a \b %Lox to a certain \e Verbosity does \e not affect
142  * the internal domains. In other words, the \e Verbosity of the internal domain
143  * (or one of its sub-domains) has to be set explicitly.
144  */

205  /** ****************************************************************************************
206  * Registers or un-registers a \b %Lox object statically with \b %ALox.
207  * Once registered, any code entity of the same process is enabled to retrieve
208  * the \b %Lox using #Get.<br>
209  * No two objects with the same name must be registered. If this is done, the latter
210  * will not be registered and not be found by #Get. In debug-compilations, an \b %ALib
211  * error report is written (by default raises 'assert') if a name is registered twice.<br>
212  * Note that name comparison is performed case <b>in</b>-sensitive.
213  *
214  * If debug-logging is enabled (depends on compilation symbols) and used, the singleton
215  * of type \% Lox provided for debug-logging is registered. This uses the name \c "Log"
216  *
217  * Registration is \e not mandatory but done by default by the constructor of class \b %Lox.
218  * Therefore, to keep a \b Lox private, an optional parameter is available.
219  *
220  * @param lox The \b %Lox to register.
221  * @param operation If \b %ContainerOp::Remove, the given \p Lox is deregistered.
222  * Defaults to \b %ContainerOp::Insert.
223  ******************************************************************************************/
