ALib Compilation Symbols

Detailed Description

Symbols (C++ preprocessor macros) listed here, support adding or removing features to ALib when compiling the library and any code unit that uses ALib. These symbols must be used only within makefiles or IDE project files to be passed to the C++ compiler (!) (normally -D option). In other words: they must not be defined or undefined within the source code that uses ALib.

ALib Symbols are prefixed with ALIB_.

For most of the symbols, a version with suffix _ON and for some also one with the suffix _OFF exists. These symbol decide about the definition, respectively the avoidance of the definition of a corresponding code selection symbol that is named the same without the suffix.
If both are set at the same time, a compiler error is generated by ALib header files. If none of them is passed to the compiler, a default value is assumed.

As a convenience, to get default behavior, no symbol needs to be passed to the compiler.

Note
Some of the compilation symbols are changing the definition of classes and methods which makes binaries incompatible with binaries that do not use the same setting for each of these symbols.
Symbols that have to be kept equal across compilation units are denoted in this documentation.

◆ ALIB_API_IS_DLL

#define ALIB_API_IS_DLL

This compiler symbol has to be defined when compiling ALib classes into a DLL under Windows/MSC. Within code units that use a DLL that contains ALib, this symbol must not be defined.

◆ ALIB_API_NO_DLL

#define ALIB_API_NO_DLL

This compiler symbol has to be defined when compiling ALib classes directly into a project Windows/MSC (not using a DLL).

See also
ALIB_API_IS_DLL and ALIB_API

◆ ALIB_AVOID_ANALYZER_WARNINGS_OFF

#define ALIB_AVOID_ANALYZER_WARNINGS_OFF

Disables certain debug code. See ALIB_AVOID_ANALYZER_WARNINGS_ON. This symbol represents the default behavior and is provided for completeness.

◆ ALIB_AVOID_ANALYZER_WARNINGS_ON

#define ALIB_AVOID_ANALYZER_WARNINGS_ON

This compiler symbol enables additional debug code that tries to prevent external analysis tools (e.g. Valgrind) to raise unnecessary warnings or errors.

When provided, it defines ALIB_AVOID_ANALYZER_WARNINGS.

◆ ALIB_DEBUG_OFF

#define ALIB_DEBUG_OFF

This compiler symbol forces ALib to compile without internal plausibility and assertion checks, by leaving the code selection symbol ALIB_DEBUG undefined. If neither this symbol nor ALIB_DEBUG_OFF is provided, then ALib headers rely on a set of standard symbols introduced (and set) by other libraries, e.g. NDEBUG.

◆ ALIB_DEBUG_ON

#define ALIB_DEBUG_ON

This compiler symbol forces ALib to compile in debug mode by defining the code selection symbol ALIB_DEBUG. Plausibility and assertion checks are performed in this mode.
If neither this symbol nor ALIB_DEBUG_OFF is provided, then ALib headers rely on the symbol NDEBUG. If NDEBUG is given, ALIB_DEBUG is not set. Otherwise it is.

◆ ALIB_DEBUG_STRINGS_OFF

#define ALIB_DEBUG_STRINGS_OFF

Disables certain debug code within class AString. See ALIB_DEBUG_STRINGS_ON. This symbol represents the default behavior and is provided for completeness.

◆ ALIB_DEBUG_STRINGS_ON

#define ALIB_DEBUG_STRINGS_ON

This compiler symbol enables additional debug code within class AString. When provided, it defines ALIB_DEBUG_STRINGS to true. This is useful when extending or specifically debugging class AString.

◆ ALIB_FEAT_THREADS_OFF

#define ALIB_FEAT_THREADS_OFF

This compiler symbol disables the use of threads within ALib, by preventing the definition of source selection symbol ALIB_FEAT_THREADS. This might be useful for platforms where no thread library is available or to minimize code size. In general, if unsure, do not use this symbol and use the multi-threaded version of ALib.
When this symbol is set, classes Thread and Runnable are not available. However, the classes ThreadLockNR and ThreadLock are! Their interface remains the same, but they do not perform any locking. Due to this behavior, it is possible to write code that performs due locking while still, in single threaded compilation, the code does not suffer any performance drawbacks, because lock code will be pruned.

◆ ALIB_FEAT_THREADS_ON

#define ALIB_FEAT_THREADS_ON

This compiler symbol enables the use of threads within ALib, by. defining source selection symbol ALIB_FEAT_THREADS. This is the default, hence the symbol is not needed to be passed, but it is available for completeness.
See ALIB_FEAT_THREADS_OFF for more information.
