Class Description

This class mimics class Thread of the Java runtime library. For general information of multi-threading support provided by ALib, see aworx::lib::threads.

§ IsAlive()

bool IsAlive ( )
inline

Returns true, if this thread was started and is still running. If false is returned, the thread object can be safely deleted, without causing a blocking operation.

Note
For system threads (those not created using ALib thread features) true is returned. It can't be determined if the thread is still alive or not.
Returns
true if this thread was started previously and is still running.

NO_DOX void Start ( )

§ Init

ALIB_API friend void Init ( )
friend

Initializes the ALib Boxing system.

The method performs the following initialization steps:

  • Adds default interface class IEquals
  • Adds default interface class IIsNull
  • Adds default interface class IIsEmpty

If code selection symbol ALIB_FEAT_BOXING_FTYPES is 1, then in addition method aworx::lib::boxing::ftypes::Init is invoked.

Initializes the default fundamental type handling of ALib Boxing. Defines interface implementation classes as interfaces of certain fundamental types.

Initializes the ALib Boxing system in respect to boxing types found in module ALib Strings.
With the use of full ALib library, this method is invoked by ALIB::Init. If ALib Strings and ALib Boxing are used as separated modules, then the user has to invoke this somewhere in the bootstrap code.

Initializes ALib thread logic. This method should not be called directly. use ALIB::Init to initialize all ALib Modules included in the distribution.

Initializes ALib time logic. This method should not be called directly. use ALIB::Init to initialize all ALib Modules included in the distribution.
