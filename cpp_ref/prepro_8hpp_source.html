19 /**
20  * @defgroup GrpALibCompilerSymbols ALib Compilation Symbols
21  *
22  * Symbols (C++ preprocessor macros) listed here, support adding or removing features to ALib when
23  * compiling the library and any code unit that uses ALib.
24  * These symbols must be used <em>only within makefiles or IDE project files to be passed to the
25  * C++ compiler (!) </em> (normally -D option). In other words: they must not be defined or undefined
26  * within the source code that uses ALib.
27  *
28  * ALib Symbols are prefixed with <em>ALIB_</em>.
29  *
30  * For most of the symbols, a version with suffix <em>_ON</em> and for some also one with the suffix
31  * <em>_OFF</em> exists. These symbol decide about the definition, respectively the avoidance of the
32  * definition of a corresponding <em>code selection symbol</em> that is named the same without the
33  * suffix.<br>
34  * If both are set at the same time, a compiler error is generated by ALib header files.
35  * If none of them is passed to the compiler, a default value is assumed.<p>
36  *
37  * As a convenience, to get default behavior, no symbol needs to be passed to the compiler.<p>
38  *
39  * \note Some of the compilation symbols are changing the definition of classes and methods which
40  * makes binaries incompatible with binaries that do not use the same setting for each of
41  * these symbols.<br>
42  * Symbols that have to be kept equal across compilation units are denoted in this
43  * documentation.
44  * @{ @}
45  */
46

64  * ALib Symbols are prefixed with <em>ALIB_</em>.<p>
