Class Description

Exception entries as stored in field Entries. The first is created with the creation of the exception itself. Further entries may be added while the exception is forwarded along the call stack using overloaded methods Add.

Hence the last entry, is probably the one with the most specific error enum element and message, while earlier entries result from "lower level" code and thus usually are more general errors.
