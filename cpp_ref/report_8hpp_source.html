60 /** ************************************************************************************************
61  * This class provides a simple facility to collect what is called a \e 'report'.
62  * Reports are maintenance messages, mostly error and warning messages, but is not aiming to replace
63  * any sort of error handling.
64  * (In \b %ALib itself, sending a \e 'report' usually precedes raising an error.)
65  *
66  * While a process can create different objects of this class, usually, the default instance
67  * available by inheriting from class
68  * \ref aworx::lib::lang::Singleton "Singleton".
69  * is sufficient and all \b %ALib internal warnings and errors will be directed to this one.
70  * Software built on \b %ALib should do the same with internal warnings and errors. An own
71  * instance might be created to collect other types of reports.
72  *
73  * This class uses a member of type
74  * \ref aworx::lib::lang::ReportWriter "ReportWriter" to actually write the reports. By default, an
75  * object of type
76  * \ref aworx::lib::lang::ReportWriterStdIO "ReportWriterStdIO" is attached.
77  *
78  * The reporting method,
79  * \ref aworx::lib::lang::Report::DoReport "DoReport" will check the flags provided with
80  * \ref aworx::lib::lang::Report::PushHaltFlags "PushHaltFlags" for message types \c 0 (errors)
81  * and \c 1 (warnings), and may invoke \e assert(). Such assertions are effective
82  * only in the debug compilation of the library/executable. Custom \e 'ReportWriters' might
83  * take action (e.g. for security reasons) and e.g. terminate the application also in
84  * release compilations.
85  *
86  * To simplify things, a set of macros is defined which are pruned in release
87  * versions of the compilation unit. These are:
88  *
89  * - #ALIB_MESSAGE
90  * - #ALIB_ERROR
91  * - #ALIB_WARNING
92  * - #ALIB_ASSERT
93  * - #ALIB_ASSERT_ERROR
94  * - #ALIB_ASSERT_WARNING
95  *
96  * In fact, these macros exist in <b>ALib Modules</b> as well, but with the inclusion of
97  * header <b>"alib/lang/report.hpp"</b>, these macros become redefined to use \b %ALib reports.
98  *
99  * For convenience, in debug compilations of the library, these macros provide the file name,
100  * line number and method name of the invocation source, which can be used by more sophisticated
101  * versions of currently attached \ref aworx::lib::lang::ReportWriter "ReportWriter".
102  *
103  * \note
104  * For debug output statements, it is advised to rather use the
105  * [ALox Logging Library](http://alexworx.github.io/ALox-Logging-Library/index.html) instead
106  * of \b %ALib reports and corresponding macros.
107  **************************************************************************************************/
