Class Description

template<typename TBoxable>
struct aworx::lib::boxing::T_Boxing< TBoxable >

This template struct is used to define customized behaviour for boxing values of type TBoxable.

If this class is not specialized for template type TBoxable, then default boxing applies. By default values and references of a type are boxed to pointers of the type, while pointer types remain such pointer types.

To specialize the class for a custom type, the following changes have to be made

  • The specialization has to be derived from std::true_type.
  • Method IsArray has to be implemented to return true if the type should be boxed as an array type
  • Type definition Type needs to use the boxed type. In case of array boxing, the array element type.
  • Method IsUnboxable has to return false if the type should not be unboxable.
  • Methods HasCustomBoxing and HasCustomUnboxing have to be implemented to return either true or false.
  • Methods Boxing and Unboxing have to be declared. The also need to be implemented if HasCustomBoxing, respectively HasCustomUnboxing return true.
See also
Macro ALIB_BOXING_SPECIALIZE which provides a simple method to defined specializations of this class.

inlinestatic

Optional custom implementation of unboxing a value.

Note
Specializations of this class usually will return type TBoxable instead of TBoxable&. The reason why this method here is declared to return a reference, is of pure technical (TMP) nature.

If method HasCustomUnboxing is specialized to return false, only the declaration of this method is needed, because namespace function DefaultUnboxing is used in this case. However, while the declaration is still necessary, the implementation is not.
