Detailed Description

The preprocessor macros listed here are the foundation for ALox Macros For Debug Logging and ALox Macros For Release Logging.

The definition of the macros is dependent on ALox Symbols To Select Code Fragments, which in turn mostly depend on ALox Compilation Symbols.

Developers that use ALox in standard use cases do not need to know or even change the macros listed here.

Macro Definition Documentation

#define ALOX_API   __declspec(dllexport)

Used to export/import C++ symbols into a dynamic link library. Defined when compiling ALox as a DLL under Windows/MSC. Dependent on ALOX_DLL_EXPORTS.

#define ALOX_CI_GET   __FILE__, __LINE__, __FUNCTION__

This macro fills in the built-in compiler symbols that provide the current source file, line number and function strings.

The macro is used by ALOX_DL_PREFIX and ALOX_RL_PREFIX

In standard use cases, there is no need to overwrite this macro. Nevertheless, it might be changed to provide other standard information into debug logging.

Note
The ALox default domain mechanism as well as the ALox code markers rely on the information provided by this macro. Changing this macro, might cause these mechanism to fail. There is no further documentation given here. Users changing this macro should be familiar with the corresponding ALox implementation details.

This is a helper macro that assembles an object named acss of type AString and fills it with debug information about the ALox compilation symbol setting.
If compilation or runtime problems occur, this macro can be used to check which symbols have been set for the compilation unit this macro is placed in.
For example, a simple code insertion into the bootstrap part of the compilation unit in question could look like this: 

{
    ALOX_COMPILER_SYMBOL_SETTINGS
    cout << "ALox compilation symbols: "    << endl;
    cout << acss.Buffer()  << endl << endl;
}
#define ALOX_DBG_LOX_INSTANCE   aworx::lox::Log::lox->

The Lox instance used by all debug logging macros. This can be overwritten (prior or after including alox.hpp) to allow different instances of class Lox for debug logging within different source entities. However, other ways to structure log output and to separate log information into different streams exists in ALox and overwriting this macro is not recommended for standard use cases.

Note: The definition must include the access operator "." or "->".

#define ALOX_DL_POSTFIX   ALOX_DBG_LOX_INSTANCE Release();

Macro that is placed at the end of almost all macros of type ALox Macros For Debug Logging. Releases the lox which was acquired when passing caller information (in macro ALOX_DL_PREFIX).

#define ALOX_DL_PREFIX   ALOX_DBG_LOX_INSTANCE AcquireAndSetCI( ALOX_CI_GET ); ALOX_DBG_LOX_INSTANCE

Macro that is placed at the beginning of almost all macros of type ALox Macros For Debug Logging. Provides caller information (provided that ALOX_DBG_LOG_CI is set) to the lox and places the debug lox instance access code using ALOX_DBG_LOX_INSTANCE.

#define ALOX_REL_LOX_INSTANCE   lox->

The Lox instance used by all release logging macros. This has to be overwritten (prior or after including alox.hpp) to provide access to the dedicated instance of class Lox created for release logging within a software.
It is of-course allowed to use different instances logging within different source entities. However, other ways to structure log output and separate log streams exists in ALox and should be evaluated prior to introducing different instances of class Lox.

Note: The definition must include the access operator "." or "->".

#define ALOX_RL_POSTFIX

Macro that is placed at the end of almost all macros of type ALox Macros For Debug Logging. Releases the lox which was acquired when passing caller information (in macro ALOX_DL_PREFIX).

#define ALOX_RL_PREFIX   ALOX_REL_LOX_INSTANCE

Macro that is placed at the beginning of almost all macros of type ALox Macros For Release Logging. Provides caller information (provided that ALOX_REL_LOG_CI is set) to the lox and places the release lox instance access code using ALOX_REL_LOX_INSTANCE.
