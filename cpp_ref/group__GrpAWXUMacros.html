Detailed Description

The preprocessor macros listed here are utility macros used by AWXU library and available to source code units that incorporate the AWXU library.

The definition of the macros is dependent on AWXU Symbols To Select Code Fragments, which in turn mostly depend on AWXU Compilation Symbols.

#define AWXU_API   __declspec(dllimport)

Used to export/import C++ symbols into a dynamic link library. Defined when compiling AWorx util classes into a DLL under Windows/MSC. Dependent on AWXU_DLL_EXPORTS.

#define AWXU_IDENTIFIER (   prefix)    AWXU_CONCAT(prefix, __LINE__)

This macro assembles an 'anonymous' identifier using the given prefix and the current line number within the source code file. This macro is used within other macros that need a C/C++ identifier internally.
As a sample, see macro OWN.

#define AWXU_ONE_TIME_WARNING (   cond,
  msg,
  identifier 
)
Value:
{ \
if( AWXU_OTW_##identifier && ( cond ) ) \
{ \
AWXU_OTW_##identifier= false; \
if( AWXU::AcquireErrorHandler() ) \
AWXU::Warning( msg ); \
} \
}

This macro, together with macros AWXU_ONE_TIME_WARNING_DECL, AWXU_ONE_TIME_WARNING_DEF and AWXU_ONE_TIME_WARNING_SET form a set of macros to implement mechanism that issues AWXU::Warning once when a certain condition is met. If multiple warnings should occur, it is possible to reset the warning flag.
In release compilation, all related code is pruned. The use of the set of macros is as follows:

  • Within the declaration of a class (normally in the header file), place the macro AWXU_ONE_TIME_WARNING_DECL. The parameter libdecl must match a macro that defines the import/export declaration of the source code entity (see AWXU_API for a sample how this identifier looks in AWXU itself). The parameter identifier can be chosen freely, and must be the same in the following steps.
  • Within a source file that holds the class definition, the macro AWXU_ONE_TIME_WARNING_DEF has to be placed.
  • Within the methods of the class that are supposed to check a condition and issue the warning once, the macro AWXU_ONE_TIME_WARNING is to be placed. This can be done more than one time, if different pieces of code should check the same or different conditions, while still only one warning should be issued.
  • Optionally, anywhere in the code (also from within methods of other classes), the macro AWXU_ONE_TIME_WARNING_SET can be used to enable (re-enable) or disable the warning.

#define OWN (   ownable)    Owner AWXU_IDENTIFIER(owner) (ownable);

This preprocessor defines an object of class Owner. The special thing about it is, that using this macro there is no need to invent an (otherwise useless) identifier for that definition. Basically, this macro exists, because C++ does not support anonymous objects.
As a sample, without using this macros a piece of code code using classes Owner/ Ownable could look like this:

{
    Owner myOwner( myOwnable );

    // do stuff
    ...
}

With the use of this macro, the code changes to:

{
    OWN( myOwnable );

    // do stuff
    ...
}
