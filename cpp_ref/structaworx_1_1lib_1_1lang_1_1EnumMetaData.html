Class Description

template<typename TEnum, typename TEnableIf = typename std::enable_if<T_EnumMetaDataDecl<TEnum>::value>::type>
struct aworx::lib::lang::EnumMetaData< TEnum, TEnableIf >

This template struct implements what we call ALib Enum Meta Data. While the TMP struct that needs to be specialized to enable enum meta data is T_EnumMetaDataDecl, this template class is the interface into it: As soon as T_EnumMetaDataDecl is specialized for type TEnum, (preferably with one of the macros found in Macros Supporting ALib Enum Types), this template class is ready to be used with the type.

The class leverages ALib Templated Singletons to provide the static meta data table, a std::vector of T_EnumMetaDataDecl<TEnum>::TTuple. Besides this, a std::map that hashes the enumeration element values to the table entries is provided.

Note
This class is not implemented to be a strict singleton. Therefore, an application might create different meta data tables for the same TEnum type. However, each of these tables will have the same data types attached, as those are defined with specialization T_EnumMetaDataDecl, unless different compilation units use different specializations. In this case, the singletons must not be accessed using GetSingleton as those are true singletons across processes.

Finally, the class implements various interface methods to easily access the meta data of enumeration elements.

See also
For a tutorial and source code sample see documentation of TMP struct T_EnumMetaDataDecl.

Member Function Documentation

◆ CheckLoad()

void CheckLoad ( )
inline

This default implementation just invokes CheckMap unless:

  • T_Resourced is specialized for TEnum and
  • the meta data tuple type is either the same as defined with macro ALIB_LANG_ENUM_NAMED or macro ALIB_LANG_ENUM_PARSABLE.

If this applies, the data is loaded from the resources.

Specialized versions might load other data types from resources or otherwise fill the Table.

If the data is provided "in code", this method does not need a specialization. Instead CheckMap has to be invoked after the insertions if it is not done by the methods that use the data.

◆ CheckMap()

void CheckMap ( bool  force = false)
inline

Clears field Map and re-inserts all elements of Table unless both containers are equally sized and force evaluates to false.

If a table contains more than one entry for the same enum element, only the first is inserted into the map. (See Enable Multiple Keywords For Parsing for more information on duplicate table rows.)

This method has to be invoked after changes of Table.

Parameters
forceIf true, the map is rebuilt, even if it has the same size as the table. Defaults to false.

◆ Get()

T_EnumMetaDataDecl<TEnum>::TTuple* Get ( TEnum  element)
inline

Searches and returns the meta data tuple for enum element element. The elements of the returned tuple can be accessed using std::get<idx>(). To receive the "original" enum element value, use static method Enum( TTuple& ).

In case T_EnumMetaDataDeclReadWrite is specialized for type TEnum, the returned tuple may be used with static method EnumReadWriteInfo::Name( TTuple& )

If no entry was found, nullptr is returned.

In debug compilations, an ALib assertion is raised, in case the map is empty.
