33 /** ************************************************************************************************
34  * A data record used to define an
35  * \ref aworx::lib::config::Variable "ALib Configuration Variable".
36  * While variables can also defined by setting their attributes individually, it is recommended to
37  * define all external configuration variables in a central place, using statically or dynamically
38  * allocated objects of this type.<br>
39  * Objects of class \b %Variable can be constructed and existing objects can be reused by invoking
40  * \ref aworx::lib::config::Variable::Define "Variable::Define". Both construction and
41  * re-definition support to pass objects of this type.
42  *
43  * All fields (except #Delim) support placeholders \c "%1", \c "%2" ... \c "%N", which are
44  * replaced with the constructor of class \b %Variable and method
45  * \ref aworx::lib::config::Variable::Define "Variable::Define".
46  * This allows to define a series of variables whose category, name, description and value is
47  * dependent on runtime information.
48  *
49  * Besides the use for passing parameters into objects of type Variable, this
50  * struct provides no further logic.
51  **************************************************************************************************/
52 struct VariableDefinition
53 {
54  /**
55  * Pointer to a category string which is used as fallback if field #Category is nulled.
56  * This is useful as usually most variables share one general application specific
57  * category. With the combination of fields #CategoryFallback and #Category,
58  * the user (of a library) is be entitled to change single variables
59  * or all at once. In the latter case, field #Category is left nulled, while this field
60  * is set to a global default value (which also can be changed, therefore a pointer).
61  */
62  String* CategoryFallback;
63 
64  /** The category for this value. May be left nulled in which case field
65  * #CategoryFallback is applied. */
66  String Category;
67 
68  /** The name of the variable. */
69  String Name;
70 
71  /**
72  * The default value. If this is \c nullptr, no variable will be created with method
73  * \ref aworx::lib::config::Configuration::Load "Configuration::Load".
74  * \note
75  * The field is ignored when using the plug-in interface
76  * \ref aworx::lib::config::ConfigurationPlugin::Load "ConfigurationPlugin::Load".
77  */
78  String DefaultValue;
79 
80  /** The delimiter passed to the plug-in as a hint on how to separate values.
81  * (May or may not be used by a plug-in) */

99 /** ************************************************************************************************
100  * This class is used to load and store external configuration data with objects of class
101  * \ref aworx::lib::config::Configuration "Configuration" and its plug-ins
102  * \ref aworx::lib::config::ConfigurationPlugin "ConfigurationPlugin".
103  *
104  * \note
105  * For general information about external configuration variables, see namespace documentation
106  * \ref aworx::lib::config "aworx::lib::config".
107  *
108  * <b>Construction/Redefinition:</b><br>
109  * While constructors accepting attributes of a variable exist, it is recommended to
110  * define all external configuration variables in a central place, using statically or dynamically
111  * allocated objects of type
112  * \ref aworx::lib::config::VariableDefinition "VariableDefinition" and pass such record
113  * to the constructor of a variable.
114  *
115  * The class is designed to be 'reused' to avoid repeated allocation/de-allocation of memory.
116  * After invoking one of the overloaded methods #Define, which share the same signatures as
117  * the overloaded constructors, a variable is freshly initialized. Internally, the memory
118  * allocated for values remains allocated.
119  *
120  * <b>Values:</b><br>
121  * A variable can contain zero, one or several values. If zero, then the variable was either not
122  * loaded, yet or the load operation failed (no configuration plug-in defined the variable).
123  * Method #Size reports the currently available values and methods #GetString(int), #GetInteger(int)
124  * and #GetFloat(int) return a value. Internally all values are stored as strings. If
125  * field #Config is set, its field
126  * \ref aworx::lib::config::Configuration::NumberFormat "Configuration::NumberFormat"
127  * is used for floating point conversion.
128  *
129  * When storing a variable that contains more than one value, field #Delim has to be set.
130  * Conversely, when reading a variable that contains multiple values, the delimiter has to be set
131  * prior to the load operation.
132  * \note
133  * This is not true when loading/storing a variable directly in a plug-in of type
134  * \ref aworx::lib::config::InMemoryPlugin "InMemoryPlugin" or might also not be true
135  * with custom configuration plug-in types which
136  * for example might store the values in a database.<br>
137  * However, with the default plug-ins
138  * \ref aworx::lib::config::CommandLinePlugin "CommandLinePlugin",
139  * \ref aworx::lib::config::EnvironmentPlugin "EnvironmentPlugin" and
140  * \ref aworx::lib::config::IniFile "IniFile"
141  * the delimiter is needed! Therefore, it is best practice to always define a proper delimiter if
142  * a variable is multi-valued.
143  *
144  * <b>Loading and Storing:</b><br>
145  * There are three ways of loading and storing a variable:
146  * - Using the interface of class \ref aworx::lib::config::Configuration "Configuration"
147  * which allows to load and store variables from different sources (plug-ins) in a prioritized
148  * way.
149  * - Using the interface of class \ref aworx::lib::config::ConfigurationPlugin "ConfigurationPlugin"
150  * which may be used if the decision about the source or drain of a load/store operation is explicitly
151  * made by a code unit.
152  * - Using the interface methods #Load,#Store, #StoreDefault and #Protect found in this class
153  * itself which are provided for convenience. Those simply invoke the interface of the static global singleton
154  * \ref aworx::lib::config::Configuration::Default "Configuration::Default".
155  *
156  * Using the interface of this class itself is the most convenient way of loading and storing
157  * variables, setting default values or protecting variables.
158  * Only very few use cases demand for the creation and use of an instance of class
159  * \b %Configuration different to the static singleton \b %Configuration::Default.
160  *
161  * Storing empty variables (method #Size returns \c 0) deletes a variable from the those
162  * configuration plug-ins that are write enabled.
163  **************************************************************************************************/
