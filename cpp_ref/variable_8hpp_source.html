33 /** ************************************************************************************************
34  * This class is used to load and store external configuration data with objects of class
35  * \ref aworx::lib::config::Configuration "Configuration" and its plug-ins
36  * \ref aworx::lib::config::ConfigurationPlugin "ConfigurationPlugin".
37  *
38  * \note
39  * For general information about external configuration variables, see namespace documentation
40  * \ref aworx::lib::config "aworx::lib::config".
41  *
42  * <b>Construction/Redeclaration:</b><br>
43  * While constructors accepting attributes of a variable exist, it is recommended to
44  * declare all external configuration variables in a central place, using statically or dynamically
45  * allocated objects of type
46  * \ref aworx::lib::config::VariableDecl "VariableDecl" and pass such record
47  * to the constructor of a variable.
48  *
49  * The class is designed to be 'reused' to avoid repeated allocation/de-allocation of memory.
50  * After invoking one of the overloaded methods #Declare, which share the same signatures as
51  * the overloaded constructors, a variable is freshly initialized. Internally, the memory
52  * allocated for values remains allocated.
53  *
54  * <b>Values:</b><br>
55  * A variable can contain zero, one or several values. If zero, then the variable was either not
56  * loaded, yet or the load operation failed (no configuration plug-in defined the variable).
57  * Method #Size reports the currently available values and methods #GetString(int), #GetInteger(int)
58  * and #GetFloat(int) return a value. Internally all values are stored as strings. If
59  * field #Config is set, its field
60  * \ref aworx::lib::config::Configuration::NumberFormat "Configuration::NumberFormat"
61  * is used for floating point conversion.
62  *
63  * When storing a variable that contains more than one value, field #Delim has to be set.
64  * Conversely, when reading a variable that contains multiple values, the delimiter has to be set
65  * prior to the load operation.
66  * \note
67  * This is not true when loading/storing a variable directly in a plug-in of type
68  * \ref aworx::lib::config::InMemoryPlugin "InMemoryPlugin" or might also not be true
69  * with custom configuration plug-in types which
70  * for example might store the values in a database.<br>
71  * However, with the default plug-ins
72  * \ref aworx::lib::config::CLIArgs "CLIArgs",
73  * \ref aworx::lib::config::Environment "Environment" and
74  * \ref aworx::lib::config::IniFile "IniFile"
75  * the delimiter is needed! Therefore, it is best practice to always define a proper delimiter if
76  * a variable is multi-valued.
77  *
78  * <b>Loading and Storing:</b><br>
79  * There are two ways of loading and storing a variable:
80  * - Using the interface of class \ref aworx::lib::config::Configuration "Configuration"
81  * which allows to load and store variables from different sources (plug-ins) in a prioritized
82  * way.
83  * - Using the interface of class \ref aworx::lib::config::ConfigurationPlugin "ConfigurationPlugin"
84  * which may be used if the decision about the source or drain of a load/store operation is
85  * explicitly made by a code unit - instead of by the configuration.
86  *
87  * Storing empty variables (method #Size returns \c 0) deletes a variable from the those
88  * configuration plug-ins that are write enabled.
89  **************************************************************************************************/
