29 /** ************************************************************************************************
30  * A logger that logs all messages to the <em>std::basic_ostream</em> instance provided in the
31  * constructor. The name of the \e Logger defaults to "ANSI_LOGGER".
32  *
33  * ALox text logger escape sequences (see class \ref aworx::lox::ESC "ESC")
34  * are translated to ANSI escape sequences.
35  * Support for ANSI escape sequences (also referred to as <em>VT100 terminal emulation</em>)
36  * is available on most unix terminal windows. Besides text colors, bold and italics font style
37  * can be set.
38  * ANSI escape sequences are also available in various IDE output windows.
39  *
40  * Foreground and background colors can be set to be either light/dark or dark/light. This improves
41  * the readability of log output a lot and even allows to read if foreground and background colors
42  * are the same (they then still differ). However, the right setting for this is dependent on
43  * the color scheme of the final output device (window). To manipulate the right setting, see field
44  * #UseLightColors and also configuration variable
45  * [ALOX_CONSOLE_LIGHT_COLORS](../group__GrpALoxConfigVars.html).
46  *
47  * In the constructor, a default format string and some other definitions in member
48  * \ref MetaInfo get set to include color settings.
49  * Of-course, these publicly accessible format attributes can be customized after creation.
50  *
51  * \note Instead of using ANSI sequences in the format strings directly, which would lower the
52  * runtime cost a little, ESC sequences are used because this way the light/dark color
53  * selection, which depends on the field #UseLightColors is performed correctly.
54  *
55  * There is not 100% match between the ANSI sequences and the definitions in
56  * \ref aworx::lox::ESC "ESC".
57  * For example ESC does not provide all ANSI colors and no blinking. On the other hand,
58  * ANSI does not allow to reset the style without resetting the colors.
59  * Of-course, it is no problem to log other ANSI codes directly into an \b %AnsiLogger.
60  * In this case, other Loggers that might be attached to the same Lox and that do not
61  * support ANSI must be equipped with corresponding replacement information.
62  * In other words: To support the same log output into different loggers, it is
63  * recommended to use \ref aworx::lox::ESC "ESC" sequences instead of
64  * directly using ANSI codes.
65  *
66  * The ANSI codes used by this class are exposed through a list of fields.
67  * They might be useful for manipulating the attributes of the \ref MetaInfo
68  * member, which of-course might contain native ANSI sequences.
69  * (In contrast to the log messages themselves, this meta information is specific to a logger
70  * instance and this way it does not need to be replaced in other loggers).
71  **************************************************************************************************/

190  /**
191  * Foreground and background colors chosen by this class might differ in their intensity.
192  * This increases the overall readability by increasing the contrast.
193  * If the background color of a console window is dark, then the background colors of
194  * colored log output should be darker colors than the foreground colors - and vice versa.
195  *
196  * Depending on the setting of this field, \b %ALox
197  * \ref aworx::lox::ESC "escape codes" for colors are translated to normal ANSI colors or
198  * lighter ones:
199  * - If this field is \ref LightColorUsage "LightColorUsage::Never", light colors are
200  * never used.
201  * - If this field is \ref LightColorUsage "LightColorUsage::ForegroundLight", foreground
202  * colors will be light colors and background colors dark ones. This is the default.
203  * - If \ref LightColorUsage "LightColorUsage::ForegroundDark", background colors will be
204  * light colors and foreground colors dark ones.
205  *
206  * The configuration variable [ALOX_CONSOLE_LIGHT_COLORS](../group__GrpALoxConfigVars.html)
207  * allows to externally modify this flag. It is read once within the constructor .
208  */
