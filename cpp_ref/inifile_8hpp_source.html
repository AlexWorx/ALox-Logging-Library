40 /** ************************************************************************************************
41  * Specialization of class #InMemoryPlugin, which reads and writes a simple configuration file
42  * consisting of sections containing key/value pairs.
43  *
44  * This class is provided for the case that no other configuration mechanism is available.
45  * In general, application specific configuration mechanisms already exist in other libraries
46  * used. Those should be adopted by creating a lean interface plug-in for \b %ALib.
47  *
48  * Some remarks on the functionality and supported format:
49  * - Comments
50  * - Comment lines at the beginning of the file are associated with the file and are written
51  * back. Such comment block is stopped with a single blank line.
52  * - Lines that start (apart from whitespace) with either a double
53  * slash \c "//", a sharp sign \c '#' or a semicolon <c>';'</c> are comment lines.
54  * - Comment lines that are added to variables in the software are using the symbol defined in
55  * filed #DefaultCommentPrefix, which defaults to \c '#'. If this is changed in the file,
56  * such changes are preserved.
57  * - Comment lines before sections and variables are associated with the respective objects
58  * and are not overwritten by comments set in the code. However, variables without
59  * comments which are overwritten in code including comments, get such comment appended.
60  * - Comments can not reside in the same line together with section names or variables.
61  * - Commented variables receive a blank line before the comment.
62  * - Commented Sections receive two blank lines before the comment. One if they are not commented.
63  *
64  * - Sections:
65  * - Sections names are enclosed by brackets \c '[' and \c ']'.
66  * - Section names can be repeated. In this case the corresponding section is continued.
67  * When the file is written, the sections are merged. Otherwise the order of sections and
68  * the variables within the section is kept intact on writing.
69  *
70  * - Variables
71  * - Variable names and their values are separated by an equal sign \c '='.
72  * - New variables inserted are formatted according to other variables found. E.g. the
73  * equal sign of all variables within a section are aligned on the same column.
74  * - Formats of variables added or changed by the user are kept intact, as long as the
75  * software does not store a value.
76  *
77  * - Continued Lines:
78  * - Variables definition are being continued (values are concatenated) if the line ends
79  * with a backslash \c '\\'.
80  * - Comment lines in-between continued lines are recognized as such. To continue a variable
81  * after a 'continued' comment line, the comment line needs to end with a backslash \c '\\'.
82  * - Variables with multiple values created (or modified) in the software, are written
83  * in continued lines, with each line showing one value, ending with the variables'
84  * delimiter character and a trailing \c '\\'.
85  * - Comment lines within Variables defined in multiple lines are removed when a variable
86  * is written back.
87  *
88  * - Escaping values
89  * - Spaces <c>' '</c> and tabulators \c '\\t' are ignored at the start and end of each line and before
90  * and after the equal sign \c '='.
91  * - Consequently, whitespaces at the start or end of a value either need to be escaped
92  * using <c>'\\ '</c> or the whole value has to be surrounded by double quotes \c ".
93  * - Values added or modified by the software that contain spaces at the start or end
94  * are surrounded by double quotes (instead of escaping them)
95  * - Double quotes in values are always escaped when writing values and have to be escaped
96  * when editing the file.
97  * - Values may consist of a list of double quoted values. Whitespaces between such
98  * values are ignored. Consequently, long strings may be enclosed in double quotes
99  * and continued in the next line when the line ends with a backslash \c '\\'.
100  * - Almost any character can be escaped. E.g \c "\a" is read as \c 'a'.
101  * - On writing only non-printable characters and double quotation marks are escaped.
102  *
103  * - Other remarks
104  * - Sequences of blank lines are reduced to one blank line, when writing the file.
105  * - Erroneous lines are ignored and not written back. Line numbers with erroneous lines
106  * are collected in field #LinesWithReadErrors.
107  **************************************************************************************************/
