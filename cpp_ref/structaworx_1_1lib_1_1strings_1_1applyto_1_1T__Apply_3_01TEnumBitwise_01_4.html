Class Description

template<typename TEnumBitwise>
struct aworx::lib::strings::applyto::T_Apply< TEnumBitwise >

Templated specialization of template struct T_Apply for enum types which meet the following conditions:

  • A specialization of TMP struct T_EnumMetaDataDeclReadWrite exists whose member NameIndex does not evalute to 0.
  • A specialization of TMP struct T_EnumIsBitwise exists that inherits from std::true_type.

(For enum types that only fulfill the first requirement, a separated specialization of T_Apply is provided.)
The conditions are evaluated by the compiler using std::enable_if on optional template parameter TEnableIf of T_Apply.

Member function Apply writes all value names corresponding to the bits set in src, separated by a comma ','. To achieve this, the static vector of tuples provided with the specialization of TMP struct T_EnumMetaDataDecl is used as a "enum element to enum name" translation table.

If the enum values of the type may be become 0 and one entry (probably the first) of the name table contains a line with 0 as enum element value, the string of this entry is written for 0 values.

Furthermore, with bitwise type enums, the name table may include enumeration values that represent combinations of more than one enum bit. Such combination entries are supported but have to be positioned in the table prior to the single bit entries! If combinational entries get matched, the corresponding single entries will not be written.

As a sample, lets consider a window manager software which has a scoped enum type representing window states. In the header file, where the enum type is defined, outside of any namespace, macro ALIB_LANG_ENUM_NAMED is used to specialize TMP struct T_EnumMetaDataDecl. Because a window might have more than one state, macro ALIB_LANG_ENUM_IS_BITWISE is used to specialize TMP struct T_EnumIsBitwise:

namespace WindowManager
{
enum class States
{
HorizontallyMaximized = (1 << 0),
VerticallyMaximized = (1 << 1),
Hidden = (1 << 2),
};
}
ALIB_LANG_ENUM_NAMED( WindowManager::States )
ALIB_LANG_ENUM_IS_BITWISE( WindowManager::States )

A window that is both, vertically and horizontally maximized, is considered to be just maximized. Therefore, we define the entries in EnumMetaData::Table that is used as a "name translation table" as follows:

EnumMetaData<WindowManager::States>::GetSingleton()->Table=
{
{ 0 , "Normal" },
// Combined entry. Put before single bits.
{ EnumValue( WindowManager::States::HorizontallyMaximized
+ WindowManager::States::VerticallyMaximized ) ,"Maximized" },
// The corresponding single bit entries.
{ EnumValue( WindowManager::States::HorizontallyMaximized ) , "HMax" },
{ EnumValue( WindowManager::States::VerticallyMaximized ) , "VMax" },
// Others...
{ EnumValue( WindowManager::States::Hidden ) , "Hidden" },
};

That is all that is needed! With this setup, the following code:

States stateNull = States(0);
States stateHM = States::HorizontallyMaximized;
States stateVM = States::VerticallyMaximized;
States stateHMVM = States::HorizontallyMaximized + States::VerticallyMaximized;
States stateHMHidden = States::HorizontallyMaximized + States::Hidden;
cout << "Null: " << stateNull << endl;
cout << "HM: " << stateHM << endl;
cout << "VM: " << stateVM << endl;
cout << "VM+HM: " << stateHMVM << endl;
cout << "HM+Hidden: " << stateHMHidden << endl;

produces this output:

Null: Normal
HM: HMax
VM: VMax
VM+HM: Maximized
HM+Hidden: HMax,Hidden

inlinestatic

Writes all enum value names corresponding to the bits set in src and the ones found in entries of specialized TMP struct member EnumMetaData::Table, separated by a comma ',' to target.

Prior to the element name list, if specialized for TEnum, a type prefix received with EnumReadWriteInfo::Prefix is written, as well as the corresponding postfix EnumReadWriteInfo::Postfix.

See class documentation for more information and a sample.
