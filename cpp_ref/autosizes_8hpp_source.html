29 /** ************************************************************************************************
30  * Auto sizes are used for tabulator positions and field sizes that expand automatically when they
31  * are exceeded. This way, any next line will have that new, expanded tabs and field sizes set
32  * leading to a nicely formatted, but still flexible, minimized width of output.
33  *
34  * To preserve the information across <em>sessions</em>, this class provides methods to transform
35  * it's information from and to string representations which can be stored in configuration files.
36  *
37  * All values are doubly stored: once for the session and once for the last imported values.
38  * Those values that were imported are used for getting the actual size. However, those that are
39  * stored for the session are the ones that would have been created if there was no import done.
40  * This leads to the fact that the session values may be smaller than the actual (imported) ones.
41  * This mechanism, in combination with the fact that on importing it can be decided if the session
42  * is newly started or not, allows the values to shrink again over different sessions.
43  **************************************************************************************************/
44 class AutoSizes
