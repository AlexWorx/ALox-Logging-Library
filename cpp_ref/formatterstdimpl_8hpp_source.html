23 /** ************************************************************************************************
24  * Base class for \b %ALib built-in formatters.
25  * This class implements abstract method
26  * \ref aworx::lib::strings::Formatter::format "Formatter::format" and introduces a set of
27  * new abstract methods that have to be implemented by descendants.
28  *
29  * The documentation of the class is split into two sections: The first addresses users of
30  * descendent classes, the second provides information on how to extend this class to implement
31  * a custom formatter.
32  *
33  * ## Documentation For Users Of Derived Classes ##
34  * The following public fields may be modified with descendent types:
35  *
36  * - Derived formatter types will set attributes in fields #DefaultNumberFormat and
37  * #AlternativeNumberFormat to reflect the default behavior of their formatting syntax.<br>
38  * Users of derived formatters may modify these attributes prior to performing a format operation
39  * to change those defaults. While this leads to a deviation of the formatting standard, it may be
40  * used instead of providing corresponding syntactic information within each and every replacement
41  * field of the format strings. Some modifications may not even be possible with just using
42  * the format specification.
43  *
44  * - Field #WriteALibErrorReports defaults to \c true and may be used to disable
45  * \ref aworx::lib::lang::Report "ALib error reports" on illegal formed format strings.
46  *
47  * - Field #WriteErrorsToTargetString defaults to \c true and may be used to disable
48  * the writing of of error messages about illegal formed format strings to the target string.
49  *
50  *
51  * ## Documentation For Implementing Derived, Custom Formatters ##
52  *
53  * Derived types need to set default values for attributes in fields
54  * #DefaultNumberFormat and #AlternativeNumberFormat within their constructor once - according
55  * to defaults specified by the formatting syntax.
56  * This should not be repeated per format operation. This way users of the type are allowed
57  * to change such default setting (even if they this may the formatter deviates from the standard
58  * it implements).
59  *
60  * All fields of this class named with prefix \c pha (for example, #phaAlignment) together comprise
61  * the set of formatting attributes which can be modified by placeholder semantics of the format string.
62  * If further attributes are needed, derived types are free to define custom members to store such
63  * attributes. Implementations provided with \b %ALib give such extended attributes the prefix
64  * \c phaExt.
65  *
66  * When parsing a placeholder of a format string, abstract method #parsePlaceholder
67  * may set field #phaFormatSpec to reflect a format-specific portion the placeholder string.
68  * If it does, it will be checked if the argument supports
69  * <b>ALib Boxing</b> interface \ref aworx::lib::strings::boxing::IFormat "IFormat"
70  * and if so, the boxing interface is invoked with this string as parameter.
71  * If the argument does not support the interface, method #parseStdFormatSpec is invoked to
72  * now parse this portion of the placeholder string in a default way.<br>
73  * This concept allows customized format specifications for custom argument types! As an example,
74  * a format specification for date/time argument types may support a custom format string
75  * like \c "yyyy-MM-dd HH:mm:ss".
76  * \note
77  * This concept is implemented with class
78  * \ref aworx::lib::strings::FormatterPythonStyle "FormatterPythonStyle" as the
79  * "Python format mini language" supports such custom format specifications. Class
80  * \ref aworx::lib::strings::FormatterJavaStyle "FormatterJavaStyle" does \b not support
81  * this mechanism.
82  *
83  * The following describes the formatting process in detail (the implementation of method #format)
84  * and this way helps to understand what is required from the implementation of the abstract methods:
85  *
86  * 1. A newline character check is made and formatting is aborted if one was found
87  * (as proposed by contract with parent class).
88  *
89  * 2. Method parameters are stored in fields #targetString, #formatString, #arguments and
90  * #argOffset. This way, the parameters can be accessed from any implemented method without
91  * the need of passing them as parameters once more.<br>
92  * In addition, field #parser is initialized. This \b Substring is used to parse
93  * the format string. Parsed portions will be consumed from the front of the string.
94  * Finally fields #argsConsumed and #nextAutoIdx are initialized to value \c 0.
95  *
96  * 3. <b>Start of the loop</b> to find and process placeholders in the format string.
97  *
98  * 4. Abstract method #findPlaceholder is invoked. If this fails (no further placeholder was found)
99  * parsing stops. If, and only if, a placeholder was found before, the rest of the string
100  * in #parser is written and abstract method #replaceEscapeSequences is invoked for this rest.
101  *
102  * 5. The portion of the format string before the placeholder position is written and abstract
103  * method #replaceEscapeSequences is invoked on this portion (if not empty).
104  *
105  * 6. Method #resetPHAs is invoked, to reset the attributes that will be parsed in the next
106  * step. The values that are set are implementation specific and need to reflect the
107  * default formatting options if no specific options are given in the format string.
108  *
109  * 7. Abstract Method #parsePlaceholder is invoked to parse and consume tokens from string #parser
110  * and while doing this, to store the parsed format attributes in the fields with
111  * name prefix \c pha (or extended attributes of a derived formatter type).<br>
112  * If an argument (positional) index is found during parsing, then method
113  * #setArgument is to be invoked by abstract method \b %parsePlaceholder providing that index.<br>
114  * If the format syntax of the formatter contains a separated format specification string
115  * (a substring of the placeholder string), then the method may store such format
116  * substring in field
117  * \ref aworx::lib::strings::FormatterStdImpl::phaFormatSpec "phaFormatSpec".
118  *
119  * 8. Next, it is checked if an argument was set by \b %parsePlaceholder. If not, #setArgument
120  * is invoked providing \c -1 for the index to indicate auto-indexing.
121  * \note
122  * If auto-indexing should be implemented differently than done with default method
123  * #setArgument, then a custom formatter might either override the method or,
124  * in the case no index is given in the format string, just
125  * set fields
126  * \ref aworx::lib::strings::FormatterStdImpl::phaArgument "phaArgument" and
127  * \ref aworx::lib::strings::FormatterStdImpl::phaArgumentIdx "phaArgumentIdx"
128  * already in \b %parsePlaceholder according to its own strategy
129  *
130  * 9. Method #preAndPostProcess is invoked with parameter \p startIdx equalling \c -1
131  * (indicating pre-processing). This allows for example to insert tab fill-characters
132  * (tab stops) prior to writing the contents of the field.
133  *
134  * 10. Method #writeCustomFormat is invoked. This allows derived formatters to write arguments in a
135  * custom way. If the method returns \c true, the loop is continued ( &rarr; Step 4.). <br>
136  * The default implementation checks if a format specification was stored in #phaFormatSpec
137  * and if yes, if boxing interface
138  * \ref aworx::lib::strings::boxing::IFormat "IFormat" is defined for #phaArgument.
139  * Un this case, the interface is invoked and \c true returned.
140  *
141  * 11. Again, if a format specification was stored in
142  * \ref aworx::lib::strings::FormatterStdImpl::phaFormatSpec "phaFormatSpec"
143  * method #parseStdFormatSpec is invoked which needs to set further attributes
144  * in the \b %Placeholder object according to to the standard format specification of the formatter.
145  *
146  * 12. Now, as all fields that represent formatting attributes (fields with prefix \c pha) are well
147  * set (or kept with their defaulted value), method #checkStdFieldAgainstArgument is invoked.
148  * This method is virtual but not abstract. Its default implementation checks the
149  * placeholder attributes against the provided argument type and raises an error if the
150  * argument does not fit to the placeholder format specification.
151  *
152  * 13. Method #writeStdArgument is invoked. This method is virtual but not abstract.
153  * Its default implementation writes the argument value formatted according to the attribute
154  * fields.
155  *
156  * 14. Finally #preAndPostProcess is invoked with parameter \p startIdx pointing to the first
157  * character in #targetString of the argument written.
158  * Here, actions like case conversion might be done on the field written.
159  *
160  * 14. End of loop ( &rarr; Step 4.)
161  *
162  **************************************************************************************************/
