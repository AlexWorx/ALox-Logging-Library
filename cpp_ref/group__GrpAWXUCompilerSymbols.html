Detailed Description

Symbols (C++ preprocessor macros) listed here, support adding or removing features to AWXU when compiling the library and any code unit that uses AWXU. These symbols must be used only within makefiles or IDE project files to be passed to the C++ compiler (!) (normally -D option). In other words: they must not be defined or undefined within the source code that uses the AWXU library.

AWXU Symbols are prefixed with AWXU_. In addition to the AWXU specific symbols, the AWXU package and headers files also support symbols that are not specific to AWXU functionality but are general symbols used in many A-Worx projects. Those symbols are prefixed AWORX_.

For most of the symbols, a version with a postfix _ON and for some also one with the postfix _OFF exists. These symbol decide about the definition, respectively the avoidance of the definition of a corresponding code selection symbol that is named the same without the postfix.
If both are set at the same time, a compiler error is generated by AWXU header files. If none of them is passed to the compiler, a default value is assumed.

As a convenience, to get default behavior, no symbol needs to be passed to the compiler.

Note
Some of the compilation symbols are changing the definition of classes and methods which makes binaries incompatible with binaries that do not use the same setting for each of these symbols.
Symbols that have to be kept equal across compilation units are denoted in this documentation.

Macro Definition Documentation

#define AWORX_GTEST_ON

This compiler symbol is passed to the compiler to define code selection symbol AWORX_GTEST which selects unit test code in accordance to the Google Test libraries. In respect to the selection of the test library, this symbol has priority over AWORX_VSTUDIO_ON.

#define AWORX_QTCREATOR_ON

This compiler symbol is passed to the compiler to define code selection symbol AWORX_QTCREATOR. This symbol should be set in all QT Creator project files which include A-Worx libraries.

#define AWORX_VSTUDIO_ON

This compiler symbol is passed to the compiler to define code selection symbol AWORX_VSTUDIO. This symbol should be set in all Microsoft Visual Studio project files which include A-Worx libraries.

Note
In respect to the selection of the unit testing framework, the symbol AWORX_GTEST_ON supersedes this symbol.
#define AWXU_DEBUG_ASTRING_ON

This compiler symbol enables additional debug code within class AString. When provided, it defines AWXU_DEBUG_ASTRING. This is useful when extending or specifically debugging class AString. This symbol may only be set in conjunction with setting AWXU_DEBUG.

#define AWXU_DEBUG_OFF

This compiler symbol forces AWXU to compile without internal plausibility and assertion checks, by leaving the code selection symbol AWXU_DEBUG undefined. If neither this symbol nor AWXU_DEBUG_OFF is provided, then the AWXU headers rely on the symbol NDEBUG: AWXU_DEBUG is set unless NDEBUG is set.

#define AWXU_DEBUG_ON

This compiler symbol forces AWXU to compile in debug mode by defining the code selection symbol AWXU_DEBUG. Plausibility and assertion checks are performed in this mode.
If neither this symbol nor AWXU_DEBUG_OFF is provided, then the AWXU headers rely on the symbol NDEBUG. If NDEBUG is given, AWXU_DEBUG is not set. Otherwise it is.

#define AWXU_DLL_EXPORTS

This compiler symbol has to be defined when compiling AWorx util classes into a DLL under Windows/MSC. Within code units that use a DLL that contains AWorx util, this symbol must not be defined.

#define AWXU_FEAT_THREADS_OFF

This compiler symbol disables the use of threads within A-Worx utility class library AWXU, by preventing the definition of source selection symbol AWXU_FEAT_THREADS. This might be useful for platforms where no thread library is available or to minimize code size. In general, if unsure, do not use this symbol and use the multi-threaded version of AWXU.
When this symbol is set, classes Thread and Runnable are not available. However, the classes ThreadLockNR and ThreadLock are! Their interface remains the same, but they do not perform any locking. Due to this behavior, it is possible to write code that performs due locking while still, in single threaded compilation, the code does not suffer any performance drawbacks, because lock code will be pruned.

#define AWXU_FEAT_THREADS_ON

This compiler symbol enables the use of threads within A-Worx utility class library AWXU, by. defining source selection symbol AWXU_FEAT_THREADS. This is the default, hence the symbol is not needed to be passed, but it is available for completeness.
See AWXU_FEAT_THREADS_OFF for more information.
