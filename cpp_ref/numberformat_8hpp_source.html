21 /** ***********************************************************************************************
22  * Defines various values needed to convert integer and floating point values
23  * to string representations and vice versa.
24  * In addition, this class provides methods that do such conversions on character array
25  * buffers.
26  *
27  * \attention
28  * The methods of this class are not intended for the common use. Instead, use interface of
29  * classes
30  * \ref aworx::lib::strings::AString "AString",
31  * \ref aworx::lib::strings::Substring "Substring" or
32  * \ref aworx::lib::strings::format::Formatter "Formatter"
33  * to write and parse numbers, which accept an object of this type as parameters.
34  *
35  * <b>Defined Singletons and User-Defined Instances:</b><br>
36  * Two static singletons of this class are defined which can be used wherever a number format object
37  * is needed as a parameter:
38  * - #Global<br>
39  * This is initialized with method
40  * \ref aworx::lib::ALIB::Init "ALIB::Init" and reflects local specific settings.
41  *
42  * - #Computational<br>
43  * This is also initialized with method
44  * \ref aworx::lib::ALIB::Init "ALIB::Init" and should be used for writing and parsing
45  * numbers which are intended to be readable by software. Its' decimal point character
46  * is set to \c '.', the international standard. Furthermore no group separators are set for
47  * decimal and decimal floating point as well as for binary, hexadecimal and octal conversion.
48  *
49  * User defined objects default to the 'computational' setting after construction.
50  *
51  * <b>Output Formats:</b><br>
52  * The following conversion formats are supported
53  *
54  * - \b Decimal<br>
55  * Supports optional minimum output width with field #DecMinimumFieldWidth,
56  * and definable thousands grouping character with field #ThousandsGroupChar, which can be
57  * activated with field #WriteGroupChars.<br>
58  * Furthermore, the plus-sign can be controlled (#PlusSign) to be either left out or be anything
59  * defined. Of-course, values \b ' ' and \b '+' are reasonable options.
60  *
61  * - <b>Binary</b><br>
62  * Binary output supports up to 64 digits and different group separators for nibbles, bytes,
63  * 16-bit words and 32 bit words. (See #BinNibbleGroupChar, #BinByteGroupChar,
64  * #BinWordGroupChar and #BinWord32GroupChar ).<br>
65  * When parsing integers, a customizable literal string defined in #BinLiteralPrefix might be used
66  * to auto-detect binary values.
67  *
68  * - <b>Hexadecimal</b><br>
69  * Hexadecimal output supports up to 16 digits (64-bit) and different group separators
70  * for bytes, 16-bit words and 32 bit words. (See #HexByteGroupChar, #HexWordGroupChar,
71  * and #HexWord32GroupChar).
72  * When parsing integers, a customizable literal string defined in #HexLiteralPrefix might be used
73  * to auto-detect hexadecimal values.
74  *
75  * - <b>Octal</b><br>
76  * Octal output supports up to 22 digits (64-bit) and different a group separator for groups
77  * of three digits defined with #OctGroupChar.
78  * When parsing integers, a customizable literal string defined in #OctLiteralPrefix might be used
79  * to auto-detect hexadecimal values.
80  *
81  * - <b>Floating Point</b><br>
82  * The width of the output is provided in two fields,
83  * #IntegralPartMinimumWidth and #FractionalPartWidth. While the integral part is
84  * a minimum width (and nothing is ever cut), the fractional part is fixed width.
85  * Values with higher fractional precision are rounded accordingly.
86  * Note that the parameter of the interface methods that may override the width, in the floating
87  * point case only affects the minimum width of the integral part.<br>
88  * The integral and fractional part of float values are separated by decimalPointChar.
89  * This field of-course has to be different from group separator #ThousandsGroupChar, which can
90  * be activated using field #WriteGroupChars.<br>
91  * Other important fields for writing and parsing floats are: #ExponentSeparator,
92  * #INFLiteral, #NANLiteral, #WriteExponentPlusSign, and #ForceScientific.
93  *
94  * <b>Notes on Writing and Parsing Values:</b><br>
95  * \attention
96  * For decimal output, the width (#DecMinimumFieldWidth) is a minimum width. This mean, that
97  * bigger numbers are written in a higher width.
98  * This is \c not true for binary, hexadecimal and octal output. In these formats, the width
99  * is an absolute value. Higher digits of numbers are just not written. The advantage of this
100  * is that no masking is needed when just the lower part of an integer number should be written.
101  * However, numbers might of-course change when cut and parsed back later!
102  *
103  * All of the integer formats have in common that the output width given is including optional
104  * grouping characters. For example if a width of \b 5 was given for decimal output, the value
105  * \c 12 would be written \c "0,012", hence \b 4 digits plus the grouping character. If grouping
106  * was disabled, the output became \c "00012", hence one digit more but the same width.
107  * In contrast to that, any sign that is written is \c not counted in the width.
108  *
109  * When parsing values, grouping characters are ignored at any position within the digits,
110  * except of the start. The same is true for whitespace characters as defined in
111  * #Whitespaces. When this field is nulled or empty, then white spaces are \b not ignored.
112  * This might be helpful in some cases where occurrence of white space characters should
113  * indicate an error (or something else) when parsing.
114  * Otherwise, the characters defined in this field are ignored at two places: at the beginning
115  * of a parsing operation and after a sign character was read.
116  *
117  * When parsing fails, a value of \c 0 (respectively \c 0.0) is returned.
118  * User-friendly classes that use the interface of this type will detect such failure
119  * through the output parameter of the parsing methods, which indicates the end of the number
120  * found.
121  *
122  * For each of the four integer formats, decimal, binary, hexadecimal and octal, dedicated
123  * parsing methods exist. Those do not accept literal prefix identifiers as defined in
124  * fields #BinLiteralPrefix, #HexLiteralPrefix and #OctLiteralPrefix. Those are
125  * identified by method #ParseInt, which aggregates the other four parsing methods.<br>
126  * There is no corresponding method defined that writes the literal prefix. When writing
127  * binary, hexadecimal or octal values, such prefixes have to be prepended 'manually'.
128  **************************************************************************************************/

544  /** ********************************************************************************************
545  * Reads an unsigned integer value in \b decimal format from the given character
546  * array at the given position.<br>
547  * Sign literals \c '-' or \c '+' are \b not accepted and parsing will fail if found.
548  * Whitespace and grouping characters, as defined in fields #Whitespaces and #ThousandsGroupChar
549  * will be ignored, regardless on their position between digits. To suppress the parsing
550  * of group characters, set field #ThousandsGroupChar to <c>'\\0'</c>. To suppress whitespace
551  * consumption before reading the value, set field #Whitespaces to nulled or empty string.
552  *
553  *
