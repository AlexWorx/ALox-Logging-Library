20 /**
21  * @defgroup GrpALoxCompilerSymbols ALox Compilation Symbols
22  *
23  * Symbols (C++ preprocessor macros) listed here, support adding or removing features to \b %ALox when
24  * compiling the library and any code unit that uses \b %ALox.
25  * These symbols must be used only within makefiles or IDE project files to be passed to the
26  * C++ compiler (normally -D option). In other words: they must not be defined or undefined within
27  * the source code that uses the \b %ALox library.
28  *
29  * For most of them, a version with a suffix "_ON" and "_OFF" exists. If both are set at the same
30  * time, a compiler error is generated by \b %ALox header files. If none of them is passed to the
31  * compiler, a default value is assumed.
32  *
33  * As a convenience, to get default behavior, no symbol needs to be passed to the compiler.
34  *
35  * \b %ALox inherits further compiler symbols from underlying library ALIB. These are documented in:
36  * \ref GrpALibCompilerSymbols.
37  *
38  * \note Some of the compilation symbols are changing the definition of classes and methods which
39  * makes binaries incompatible with binaries that do not use the same setting for each of
40  * these symbols.<br>
41  * Symbols that have to be kept equal across compilation units are denoted in this
42  * documentation.
43  *
44  * @{ @}
45  */
46 
47 /**
48  * @defgroup GrpALoxCodeSelectorSymbols ALox Symbols to Select Code Fragments
49  *
50  * Symbols (C++ preprocessor macros) listed here, are mostly generated out of
51  * \ref GrpALoxCompilerSymbols. In this case, their name corresponds to these symbols, excluding
52  * the suffixes "_ON" and "_OFF".
53  *
54  * These symbols must not be passed to the compiler and must also not be set manually from within
55  * code entities that use \b %ALox libraries. They can be used in code entities that use ALox to select
56  * code depending on the same settings of the \b %ALox library compilation. Selection takes place,
57  * using preprocessor directives:
58  *
59  * \code{.cpp}
60  * #if defined(ALOX_XYZ_SELECTOR)
61  * // your selective code
62  * #endif
63  * \endcode
64  *
65  * \b %ALox inherits further compiler symbols from underlying library ALIB. These are documented in:
66  * \ref GrpALibCodeSelectorSymbols.
67  *
68  * (Besides \b %ALox specific symbols of that type, \b %ALox internally uses also platform specific symbols
69  * to select code, e.g. \_\_GLIBCXX\_\_, _MSC_VER, etc.).
70  * @{ @}
71  */

80 /**

82  * @{ \def ALOX_DBG_LOG_ON
83  * Explicitly enables debug logging by defining the code selection symbol \ref ALOX_DBG_LOG.
84  *
85  * Must not be used in conjunction with \ref ALOX_DBG_LOG_OFF.
86  *
87  * If neither <b>ALOX_DBG_LOG_ON</b> nor <b>ALOX_DBG_LOG_OFF</b> are specified, \b %ALox will determine the
88  * inclusion of debug <em>Log Statements</em> by evaluating macro #ALIB_DEBUG.
89  * If ALIB_DEBUG is set, ALOX_DBG_LOG will also be set.
90  *
91  * @}
92  *
93  * @addtogroup GrpALoxCompilerSymbols
94  * @{ \def ALOX_DBG_LOG_OFF
95  * Explicitly disables debug logging by leaving the code selection symbol \ref ALOX_DBG_LOG undefined.
96  * Any debug <em>Log Statements</em> that are embedded in code by using the \ref GrpMacrosDebugLog, will
97  * get pruned.
98  *
99  * Must not be used in conjunction with \ref ALOX_DBG_LOG_ON.
100  *
101  * If neither <b>ALOX_DBG_LOG_ON</b> nor <b>ALOX_DBG_LOG_OFF</b> are specified, \b %ALox will determine the
102  * inclusion of debug <em>Log Statements</em> by evaluating macro #ALIB_DEBUG.
103  * If ALIB_DEBUG is set, ALOX_DBG_LOG will also be set.
104  * @}
105  *
106  * @addtogroup GrpALoxCodeSelectorSymbols
107  * @{ \def ALOX_DBG_LOG
108  * If defined as \c true, debug <em>Log Statements</em> are enabled. Hence, users of \b %ALox can
109  * conditionally compile special debug logging code that normally belongs to corresponding
110  * debug logging statements.
111  * This is useful for example, to conditionally compile code that calculates and gathers
112  * information to do some more complex log output.
113  *
114  * See \ref ALOX_DBG_LOG_ON and \ref ALOX_DBG_LOG_OFF for information about how to manipulate
115  * this symbol.
116  * @}
117  *
118 */

213 /**
214  * @addtogroup GrpALoxCompilerSymbols
215  * @{ \def ALOX_DBG_LOG_CI_ON
216  * Explicitly enables the generation of scope information using the built-in preprocessor
217  * macros like "__FILE__" and "__LINE__" for \b %ALox debug logging statements
218  *
219  * Passing this Symbol to the compiler, makes \b %ALox define the code selection symbol
220  * \ref ALOX_DBG_LOG_CI.
221  * Must not be used in conjunction with \ref ALOX_DBG_LOG_CI_OFF.
222  *
223  * If neither <b>ALOX_DBG_LOG_CI_ON</b> nor <b>ALOX_DBG_LOG_CIO_OFF</b> are specified, \b %ALox will enable
224  * caller source information for debug logging. In other words, <b>ALOX_DBG_LOG_CI_ON</b> is the
225  * default and passing the symbol to the compiler is not mandatory.
226  *
227  * \note The \b %ALox <em>Scope Domain</em> mechanism as well as the ALox features \e Lox.Once and
228  * <em>Log Data</em> rely on caller information.
229  * These mechanisms are therefore not available to debug logging unless this symbol is set.
230  * @}
231  *
232  *
233  * @addtogroup GrpALoxCompilerSymbols
234  * @{ \def ALOX_DBG_LOG_CI_OFF
235  * Disables the generation of scope information using the built-in preprocessor
236  * macros like "__FILE__" and "__LINE__" for \b %ALox debug logging statements
237  *
238  * Passing this Symbol to the compiler, makes \b %ALox to leave the code selection symbol
239  * \ref ALOX_DBG_LOG_CI undefined.<br>
240  * Must not be used in conjunction with \ref ALOX_DBG_LOG_CI_ON.
241  *
242  * If neither <b>ALOX_DBG_LOG_CI_ON</b> nor <b>ALOX_DBG_LOG_CIO_OFF</b> are specified, then
243  * \ref ALOX_DBG_LOG_CI_ON is the default.
244  *
245  * \note The \b %ALox Scope Domain mechanism as well as the ALox features \e Lox.Once and

309  * in the debug version of a compilation unit.
