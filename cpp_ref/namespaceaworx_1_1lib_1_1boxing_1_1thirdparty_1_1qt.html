aworx::lib::boxing::thirdparty::qt Namespace Reference

Namespace Description

With the inclusion of header file alib/compatibility/qt.hpp, support for boxing string types of the QT Class Library is provided. QT provides several string types, the most prominent of-course is QString. Unfortunately, this type uses a 2-byte character type. This is neither compatible with C++ fundamental type char nor with wchar_t, as the latter has platform dependent size, e.g. is 4 bytes wide on GNU/Linux using GNU C++ compiler. QString is boxed to type char16_t[].

Besides this, supported types are QByteArray and QLatin1String. Those types are boxed to fundamental type char[] (the same as for example aworx::String and std::string are), and therefor can be used completely transparently with any function or method that accepts boxes and processes single byte string values.

All types are unboxable as values.

For more information see chapter Boxing Strings of the namespace documentation of module ALib Boxing.

Note
Namespace function aworx::lib::boxing::InitQTString is to be invoked on bootstrap to complete the support of QT-string-type boxing.
